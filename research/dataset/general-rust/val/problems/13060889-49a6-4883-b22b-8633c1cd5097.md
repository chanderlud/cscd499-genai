# Cheapest Route With One Coupon

You are given a **directed** graph with `n` nodes labeled `0..n-1`. Each directed edge has a positive integer cost.

You want to travel from node `0` to node `n-1` with minimum total cost, with one twist:

- You may use **at most one coupon** on **at most one edge** along your route.
- If you apply the coupon to an edge of cost `w`, that edge costs `floor(w / 2)` instead.
- You may also choose to **not** use the coupon at all.

If node `n-1` is unreachable from node `0`, return `None`.

## Function Signature

```rust
pub fn min_cost_with_coupon(n: usize, edges: &[(usize, usize, i64)]) -> Option<i64>
````

* `n`: number of nodes
* `edges`: a list of `(from, to, cost)` directed edges

## Requirements

* Nodes are `0..n-1`
* `cost > 0`
* Multiple edges between the same pair of nodes may exist.
* Return the minimum total cost using the coupon **at most once**, or `None` if unreachable.

## Constraints (for intended solution)

* `1 <= n <= 200_000`
* `0 <= edges.len() <= 200_000`
* `1 <= cost <= 10^12`

Your solution should be efficient (typically `O((n + m) log n)`).

## Example

If `n = 2` and `edges = [(0, 1, 10)]`, the best route is to use the coupon on the only edge:

* cost becomes `floor(10 / 2) = 5`
* answer is `Some(5)`
