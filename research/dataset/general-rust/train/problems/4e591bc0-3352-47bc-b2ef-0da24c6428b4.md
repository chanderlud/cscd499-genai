## Problem: Two Sum (Indices)

### Function Signature (Rust)

```rust
pub fn two_sum(nums: Vec<i32>, target: i32) -> Option<(usize, usize)>;
```

### Requirements

You are given an array of integers `nums` and an integer `target`. Return **indices** of the **two distinct** elements in `nums` such that they add up to `target`.

* If such a pair exists, return `Some((i, j))` where:

    * `i` and `j` are valid indices into `nums`
    * `i != j`
    * `nums[i] + nums[j] == target`
* If no such pair exists, return `None`.
* You may return the indices in **any order**.
* Each input will have **at most one valid answer**.

### Constraints

* `2 <= nums.len() <= 100_000`
* `-1_000_000_000 <= nums[i] <= 1_000_000_000`
* `-1_000_000_000 <= target <= 1_000_000_000`

### Performance Expectations

* Time: **O(n)** expected
* Space: **O(n)** allowed
* Don’t do the obvious **O(n²)** thing unless you enjoy wasting electricity.

### Examples

* `nums = [2, 7, 11, 15], target = 9` → `Some((0, 1))`
* `nums = [3, 2, 4], target = 6` → `Some((1, 2))`
* `nums = [3, 3], target = 6` → `Some((0, 1))`
* `nums = [1, 2, 3], target = 7` → `None`

### Notes / Edge Cases to Handle

* Duplicate values (like `[3, 3]`)
* Negative numbers
* Large input size (so don’t sort unless you’re returning values instead of indices)
