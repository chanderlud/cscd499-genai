- **Spec:**
    - Implement a function that inspects an ordered slice of magical `Rune` values and finds the **longest contiguous spell window** whose total `cost` does not exceed a given `budget` and whose `glyph` names are all unique within that window.
    - Return a `SpellWindow` that **borrows** the winning sub-slice from the original input.
    - If multiple windows have the same length, return the one with the **smallest total cost**.
    - If there is still a tie, return the one that appears **earliest** in the input.
    - Validate the input before searching:
        - a budget of `0` is an error
        - any rune with an empty glyph is an error
    - If no rune can fit inside the budget, return an empty borrowed window with total cost `0`.

- **Constraints:**
    - The solution must be deterministic and unit testable.
    - The returned window must borrow from the input slice, not allocate copied runes.
    - Use Rust type safety with explicit structs and a custom error enum.
    - The implementation should naturally exercise iterators, lifetimes, and error handling.
    - Assume each rune cost is a non-negative `u32`.

- **Signature:**
  ```rust
  pub fn best_spell_window<'a>(
      runes: &'a [Rune<'a>],
      budget: u32,
  ) -> Result<SpellWindow<'a>, SpellError>
  ```
- **Example:**
  ```rust
let runes = [
Rune { glyph: "sun", cost: 2 },
Rune { glyph: "mist", cost: 3 },
Rune { glyph: "sun", cost: 1 },
Rune { glyph: "ember", cost: 2 },
];

let window = best_spell_window(&runes, 5).unwrap();
assert_eq!(window.total_cost, 4);
assert_eq!(window.runes.iter().map(|r| r.glyph).collect::<Vec<_>>(), vec!["sun", "ember"]);
  ```
