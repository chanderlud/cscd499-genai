- **Spec:** Write a function that reads a slice of log lines shaped like `name|count|delta`. Trim each line and ignore empties. Return `Vec<Beacon>` preserving input order. Each `Beacon` must borrow `name` from the input and store `count`, `delta`, and `power = count * abs(delta)`. Reject malformed rows, duplicate names, invalid names, or arithmetic overflow.
- **Constraints:** `name` must contain only ASCII lowercase letters or `-`, and be non-empty. `count` is `u32`. `delta` is `i32`. Use iterator-based parsing where practical. Return a custom error enum that borrows the offending line or name. No allocation for names beyond what parsing already needs.
- **Signature:**
  ```rust
  pub fn parse_beacon_log<'a>(lines: &'a [&'a str]) -> Result<Vec<Beacon<'a>>, LogError<'a>>
  ```
- **Example:**
  ```rust
let rows = ["north-lamp|3|-4", "", "east|2|5"];
let out = parse_beacon_log(&rows).unwrap();
assert_eq!(out[0].power, 12);
assert_eq!(out[1].name, "east");
  ```
