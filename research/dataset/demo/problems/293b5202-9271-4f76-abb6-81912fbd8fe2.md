# Problem: Chrono-Transit (Time-Dependent Shortest Route + Counting + Canonical Path)

You are given a directed graph of `n` stations (0-indexed) and a list of directed edges.
Time starts at `0` at `source`.

Each edge is a *scheduled connection* described by:
`(from, to, travel_time, offset, interval)`

When you are at node `from` at time `t`, you may traverse the edge by waiting until the
next departure time `d >= t`, then arriving at `to` at time `d + travel_time`.

Departure time rules:
- If `interval == 0`: the edge departs immediately at time `d = t` (no schedule, no waiting).
- If `interval > 0`: departures occur at times `offset + k*interval` for integers `k >= 0`,
  and you must take the smallest such departure `d` with `d >= t`.

Formally:
```text
if interval == 0:
    d = t
else if t <= offset:
    d = offset
else:
    d = offset + ceil((t - offset) / interval) * interval
```

The *route time* is the final arrival time at `target`.
The *total waiting time* of a route is the sum, over each traversed edge, of `(departure_time - arrival_time_at_from)`.
The *rides* count is the number of edges in the route.

Your task is to implement `chrono_transit` to compute:
1) The earliest possible arrival time at `target`.
2) The number of distinct routes achieving that earliest arrival time, modulo `1_000_000_007`.
3) A single **canonical** optimal route (as a vector of node indices) chosen among all
   earliest-arrival routes by the following tie-breakers (in order):
   - minimize number of rides (edges)
   - if still tied, minimize total waiting time
   - if still tied, choose the lexicographically smallest node sequence

Lexicographic order is the usual: compare the first position where they differ; the smaller node id wins.
(If one is a prefix of the other, the shorter sequence is lexicographically smaller.)

Return `None` if `target` is unreachable.

## Constraints (designed to ruin your weekend)
- `1 <= n <= 200_000`
- `0 <= edges.len() <= 400_000`
- Node ids are in `0..n`
- `travel_time >= 1` (guarantees time strictly increases along any edge traversal, so the number of optimal routes is finite)
- `0 <= offset <= 10^18`
- `0 <= interval <= 10^12`
- Start time at `source` is exactly `0`

## Correctness requirements
- Must not overflow (use 128-bit intermediates where needed).
- Must be deterministic.
- Must handle `interval == 0` exactly as specified.
- Must return the canonical optimal route as defined.
