# Problem: Merge k Sorted Streams with Dedup + Coverage

You are given `k` sorted integer streams. Each stream is represented as a sorted `Vec<i64>` (non-decreasing order). Streams may contain duplicates internally, and different streams may share values.

Your task is to merge these streams into a single sorted output **without duplicates**, while also computing a *coverage count* for each output value: how many distinct streams contained that value at least once.

Return a vector of pairs `(value, coverage)` in ascending order by `value`.

## Function Signature (Rust)

```rust
pub fn merge_unique_with_coverage(streams: Vec<Vec<i64>>) -> Vec<(i64, usize)>;
````

## Requirements

1. **Sorted input:** Each stream is sorted in non-decreasing order.
2. **Dedup semantics:**

    * The output must contain each value at most once.
    * A stream contributes at most 1 to coverage for a given value, even if it contains the value multiple times.
3. **Coverage semantics:**

    * `coverage` is the number of **distinct streams** that contain the value at least once.
4. **Output ordering:** Output pairs must be sorted strictly increasing by `value`.
5. **Complexity target:**

    * Aim for `O(N log k)` time where `N` is the total number of elements across all streams.
    * Extra memory should be `O(k)` (excluding the output).

## Notes / Hints

* A classic approach uses a min-heap of the current head element from each stream.
* Be careful:

    * You must skip duplicates within a stream efficiently.
    * When multiple streams currently point at the same value, you should count them once each, then advance each of those streams past that value (and past duplicates).
