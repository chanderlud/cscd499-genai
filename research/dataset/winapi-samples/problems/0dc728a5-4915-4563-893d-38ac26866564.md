1) Named Pipe IPC Server and Client

**Spec:** Implement a server that creates a named pipe and listens for incoming commands, and a client that sends a command to the server. The server must parse the incoming string into an `IpcCommand` enum (with variants `Toggle`, `Show`, `Hide`, `Quit`) and send it via a channel. The client must send a string command and return success. The server must ignore the string "ping" and break the server loop when receiving "quit" or "exit".

**Constraints:**
- Use the Win32 named pipe API (CreateNamedPipeW, ConnectNamedPipe, ReadFile, WriteFile, etc.)
- The server must run in a separate thread and return a `std::sync::mpsc::Receiver<IpcCommand>`.
- The client function must take a string command and return a boolean.
- The pipe name must be `\\.\pipe\app-launcher-ipc`.

**Signature:**
```rust
pub fn start_ipc_server() -> mpsc::Receiver<IpcCommand>;
pub fn send_command(cmd: &str) -> bool;
```

**Example:**
```rust
let rx = start_ipc_server();
// In another process or thread:
send_command("toggle");
// The server thread will receive IpcCommand::Toggle via the channel.
```
