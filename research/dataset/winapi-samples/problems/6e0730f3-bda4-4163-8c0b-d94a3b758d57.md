N) Setting Focus to Child Controls and Routing Keyboard Input

**Spec:** Create a Win32 window application with an edit control and a list box that allows keyboard navigation between them using the Down and Up arrow keys. When the Down arrow is pressed in the edit control, focus moves to the list box and selects the first item. When the Up arrow is pressed in the list box, focus moves back to the edit control.

**Constraints:** 
- Use the `windows` crate for Win32 API bindings.
- The main window must be created with the class name "FocusExample".
- The edit control must be 36 pixels high and span the width of the main window.
- The list box must fill the remaining space below the edit control and have a vertical scroll bar.
- The child controls must be subclassed to intercept keyboard messages.
- The application must handle the `WM_CREATE`, `WM_PAINT`, and `WM_DESTROY` messages appropriately.

**Signature:** The solution must include the following functions (or equivalent):
```rust
fn route_nav_keys_to_list_box(list_box: HWND, msg: u32, wparam: WPARAM, lparam: LPARAM) -> Option<LRESULT>;
fn route_key_up_to_text_box(text_box: HWND, msg: u32, wparam: WPARAM, lparam: LPARAM) -> Option<LRESULT>;
```

**Example:**
```rust
// In the text box window procedure:
if let Some(result) = route_nav_keys_to_list_box(list_box_hwnd, msg, wparam, lparam) {
    return result;
}

// In the list box window procedure:
if let Some(result) = route_key_up_to_text_box(text_box_hwnd, msg, wparam, lparam) {
    return result;
}
```
