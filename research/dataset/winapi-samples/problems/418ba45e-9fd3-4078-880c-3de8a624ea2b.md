1) Pass Data to Window Procedure

**Spec:** Create a Win32 window that passes a custom data structure to the window procedure via the `lpCreateParams` parameter of `CreateWindowExW`. The window procedure must store this data in the window's user data area (using `SetWindowLongPtrW` with `GWLP_USERDATA`) and retrieve it when needed. The data must be properly cleaned up when the window is destroyed.

**Constraints:**
- Use the `windows` crate for Win32 bindings.
- The window class must be registered with a custom window procedure.
- The window must be created with `WS_OVERLAPPEDWINDOW` and `WS_VISIBLE` styles.
- The custom data must be passed via `lpCreateParams` and stored in the window's user data.
- The window procedure must handle `WM_CREATE` to store the data and `WM_DESTROY` to clean it up.

**Signature:**
```rust
struct WindowData {
    message: String,
}

extern "system" fn window_proc(hwnd: HWND, msg: u32, wparam: WPARAM, lparam: LPARAM) -> LRESULT;
fn main() -> windows::core::Result<()>;
```

**Example:**
```rust
// In window_proc for WM_CREATE:
let cs = lparam.0 as *const CREATESTRUCTW;
let data_ptr = (*cs).lpCreateParams as *mut WindowData;
let data = Box::from_raw(data_ptr);
SetWindowLongPtrW(hwnd, GWLP_USERDATA, Box::into_raw(data) as _);

// In main:
let window_data = Box::new(WindowData { message: String::from("Hello!") });
let hwnd = CreateWindowExW(
    WINDOW_EX_STYLE(0),
    window_class,
    w!("Data Passing Example"),
    WS_OVERLAPPEDWINDOW | WS_VISIBLE,
    CW_USEDEFAULT, CW_USEDEFAULT, 400, 300,
    None, None, Some(instance),
    Some(Box::into_raw(window_data) as _),
)?;
```
