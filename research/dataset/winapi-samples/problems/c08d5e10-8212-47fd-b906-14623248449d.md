Retrieve Button Image Type and Handle

**Spec:** Create a Win32 window application that creates a button with an icon and uses the `BM_GETIMAGE` message to determine the type of image (bitmap or icon) on the button, then display the result in a message box.

**Constraints:**
- Use the `windows` crate for Win32 bindings.
- The button must be created with the `BS_ICON` style and have an icon set via `BM_SETIMAGE`.
- The application must handle the `WM_CREATE` message to create the button, set the icon, and then use `BM_GETIMAGE` to check for both `IMAGE_BITMAP` and `IMAGE_ICON`.

**Signature:**
```rust
fn main() -> windows::core::Result<()>
```

**Example:**
```rust
// In the WM_CREATE handler:
let button_hwnd = CreateWindowExW(/* ... */, "BUTTON", /* ... */, BS_ICON as u32 | WS_VISIBLE | WS_CHILD, /* ... */);
let icon = LoadIconW(None, IDI_APPLICATION).unwrap();
SendMessageW(button_hwnd, BM_SETIMAGE, WPARAM(IMAGE_ICON.0 as usize), LPARAM(icon.0 as isize));

let bitmap_result = SendMessageW(button_hwnd, BM_GETIMAGE, WPARAM(IMAGE_BITMAP.0 as usize), LPARAM(0));
let icon_result = SendMessageW(button_hwnd, BM_GETIMAGE, WPARAM(IMAGE_ICON.0 as usize), LPARAM(0));

if bitmap_result.0 != 0 {
    MessageBoxW(/* ... */, "Button has a bitmap image", /* ... */, MB_OK);
} else if icon_result.0 != 0 {
    MessageBoxW(/* ... */, "Button has an icon image", /* ... */, MB_OK);
} else {
    MessageBoxW(/* ... */, "Button has no image", /* ... */, MB_OK);
}
```
