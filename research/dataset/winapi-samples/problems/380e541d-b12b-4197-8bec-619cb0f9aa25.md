1) Get Window Info via UI Automation

**Spec:** Write a function that uses the Windows UI Automation API to retrieve the current name and executable file name of a window given its title.

**Constraints:**
- Use the `windows` crate and the `Win32::UI::Accessibility` and `Win32::System::Com` modules.
- Initialize COM with `COINIT_MULTITHREADED` before any UI Automation operations.
- Find the window by its exact title (case-sensitive) using `FindWindowA`.
- Use the COM interface `IUIAutomation` to get the element from the window handle.
- Retrieve the current name of the element via the COM interface.
- Attempt to cast the COM element to the WinRT `AutomationElement` to retrieve the executable file name. If the cast fails (on older Windows versions), return `None` for the executable file name.
- Return the window name and the optional executable file name as a tuple.

**Signature:**
```rust
fn get_window_info(window_title: &str) -> Result<(String, Option<String>)>
```

**Example:**
```rust
// Assuming the Calculator app is running with the exact title "Calculator"
let (name, exe) = get_window_info("Calculator")?;
println!("Window name: {name}, Executable: {exe:?}");
```
