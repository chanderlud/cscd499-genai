1) Query Memory Region

**Spec:** Write a function that queries the memory region information for a given address in the current process using the Win32 API `VirtualQuery`. Then, in `main`, use this function to retrieve and display the memory region details for the base address of the current module.

**Constraints:**
- Use `VirtualQuery` to retrieve the memory information.
- The function must return a `Result` containing `MEMORY_BASIC_INFORMATION` on success.
- Handle errors by returning a `windows::core::Error` when `VirtualQuery` fails.
- Use `GetModuleHandleW` to obtain the base address of the current module.

**Signature:**
```rust
fn query_memory_region(addr: usize) -> Result<MEMORY_BASIC_INFORMATION>
```

**Example:**
```rust
fn main() -> Result<()> {
    let module: HMODULE = unsafe { GetModuleHandleW(None) }?;
    let module_addr = module.0 as usize;
    let memory_info = query_memory_region(module_addr)?;
    println!("Memory region at base address {:#x}:", module_addr);
    println!("  Base address: {:p}", memory_info.BaseAddress);
    println!("  Allocation base: {:p}", memory_info.AllocationBase);
    println!("  Region size: {} bytes", memory_info.RegionSize);
    println!("  State: {:?}", memory_info.State);
    println!("  Protect: {:?}", memory_info.Protect);
    Ok(())
}
```
