1) Undecorated Resizing Window

**Spec:** Implement a window procedure and a hit test function that enable an undecorated child window to be resized by dragging its borders.

**Constraints:**
- Use the `windows` crate for Win32 API bindings.
- The window procedure must handle `WM_CREATE` to store user data, `WM_NCHITTEST` to return the appropriate hit-test code, `WM_NCLBUTTONDOWN` to forward the resize message to the parent, and `WM_NCDESTROY` to clean up.
- The hit test function must use a fixed border width and height (8 pixels) to determine if the mouse is over a border or corner.
- The window must be a child window and must check the parent's style for `WS_SIZEBOX` to determine if it is resizable.
- The hit test function must return an enum `HitTestResult` that can be converted to Win32 hit-test codes (like `HTLEFT`, `HTRIGHT`, etc.).

**Signature:**
```rust
unsafe extern "system" fn drag_resize_window_proc(
    hwnd: HWND,
    msg: u32,
    wparam: WPARAM,
    lparam: LPARAM,
) -> LRESULT;

fn hit_test(
    left: i32,
    top: i32,
    right: i32,
    bottom: i32,
    cx: i32,
    cy: i32,
    border_x: i32,
    border_y: i32,
) -> HitTestResult;
```

**Example:**
```rust
// In the window procedure, for WM_NCHITTEST:
let mut rect = RECT::default();
GetWindowRect(hwnd, &mut rect)?;
let (cx, cy) = (GET_X_LPARAM(lparam), GET_Y_LPARAM(lparam));
let res = hit_test(rect.left, rect.top, rect.right, rect.bottom, cx, cy, 8, 8);
return LRESULT(res.to_win32() as _);
```
