Fuzzy Search Window

**Spec:** Create a Win32 popup window application that provides fuzzy search functionality over a predefined list of strings. The window must contain an editable text box at the top and a list box below it. As the user types in the text box, the list box should update in real-time to show only items that fuzzy-match the input, sorted by match score. The user must be able to navigate the list with arrow keys and select an item with Enter, which prints the selection to the console and exits the application. Pressing Escape must also exit.

**Constraints:**
- Use the `windows` crate for Win32 API bindings.
- The main window must be a popup (`WS_POPUP`) with dimensions 600x400.
- The text box must be an Edit control (`"Edit"` class) with height 36 pixels.
- The list box must be a ListBox control (`"ListBox"` class) with `LBS_NOTIFY` and `WS_VSCROLL` styles.
- Implement a custom fuzzy matching algorithm that is case-insensitive and assigns higher scores for character matches at the same index.
- Subclass both the text box and list box to handle keyboard navigation (arrow keys, Enter, Escape) and message routing.
- The predefined list of strings must be: `["Notepad", "Calculator", "Command Prompt", "File Explorer", "Task Manager"]`.

**Signature:**
```rust
fn main() -> windows::core::Result<()>
```

**Example:**
```rust
// Running the application opens a window.
// Typing "calc" in the text box filters the list to show "Calculator".
// Pressing Enter selects "Calculator", prints "Selected: Calculator" to the console, and exits.
```
