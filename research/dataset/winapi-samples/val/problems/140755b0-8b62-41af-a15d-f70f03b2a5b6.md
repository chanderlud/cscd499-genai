**Spec:** Implement ECDH key agreement using BCrypt to derive a shared secret between two parties. The function should:
1. Generate an ephemeral ECDH key pair for curve 25519 (256-bit key length) using the provided algorithm handle
2. Export the public key as a blob in BCRYPT_ECCPUBLIC_BLOB format
3. Import the peer's public key blob in BCRYPT_ECCPUBLIC_BLOB format
4. Derive the raw shared secret using the ephemeral private key and peer's public key via BCryptSecretAgreement
5. Return both the generated public key blob and the derived shared secret

**Constraints:**
- Use `BCryptGenerateKeyPair`, `BCryptFinalizeKeyPair`, `BCryptExportKey`, `BCryptImportKeyPair`, `BCryptSecretAgreement`, and `BCryptDeriveKey`
- The algorithm handle is pre-initialized for ECDH with curve 25519
- Clean up all BCrypt handles (key handles, secret handle) using `BCryptDestroyKey`/`BCryptDestroySecret`
- Use `BCRYPT_ECCPUBLIC_BLOB` format for key export/import
- Use `BCRYPT_KDF_RAW_SECRET` for key derivation
- Return the raw shared secret bytes without additional processing
- Convert NTSTATUS errors to windows::core::Result using HRESULT

**Signature:**
```rust
fn perform_ecdh_key_agreement(
    alg_handle: BCRYPT_ALG_HANDLE,
    peer_public_key_blob: &[u8],
) -> windows::core::Result<(Vec<u8>, Vec<u8>)>
```

**Example:**

// Assume alg_handle is already initialized for ECDH with curve 25519
let peer_blob = vec![/* peer's public key blob */];
let (my_public_blob, shared_secret) = perform_ecdh_key_agreement(alg_handle, &peer_blob)?;
