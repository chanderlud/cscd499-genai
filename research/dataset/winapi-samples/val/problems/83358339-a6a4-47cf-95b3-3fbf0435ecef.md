**Spec:** Write a function that reads a file system path string from the Windows registry, then uses that path to create an `IShellItem` and write its normalized `SIGDN_FILESYSPATH` representation into a provided buffer without heap allocation for the final output. The registry data is read as a wide string and validated for absolute path and length (≤260 characters including null terminator).

**Constraints:**
- First, read a `REG_SZ` value from the specified registry hive, key path, and value name using Win32 registry APIs (W-suffix).
- The registry data is read into a wide string buffer (`Vec<u16>`). Validate that the path is absolute (drive letter `C:\` or UNC `\\`) and no longer than 260 characters (including null terminator) by examining the wide characters.
- Use the retrieved wide string to create an `IShellItem` via `SHCreateItemFromParsingName`.
- Call `IShellItem::GetDisplayName` with `SIGDN_FILESYSPATH` and copy the resulting null-terminated wide string into the provided output buffer.
- The output buffer is a mutable slice of `u16`; the function must write the string and return the number of characters (excluding null terminator) written.
- If the output buffer is too small, the function returns an error.
- Do not allocate on the heap for the final output copy (no `Vec`, `String`, `PathBuf`, etc. for that step). Intermediate allocations for registry reading are permitted.
- Use W-suffix Win32 APIs where applicable.

**Signature:**
```rust
pub fn registry_path_to_shellitem_path(
    hive: HKEY,
    key_path: &str,
    value_name: &str,
    output: &mut [u16],
) -> windows::core::Result<usize>
```

**Example:**
```rust
let mut buffer = [0u16; 260];
let len = registry_path_to_shellitem_path(
    HKEY_LOCAL_MACHINE,
    "SOFTWARE\\Microsoft\\Windows NT\\CurrentVersion",
    "SystemRoot",
    &mut buffer,
)?;
// buffer[..len] now contains the wide characters of the normalized system root path.
```
