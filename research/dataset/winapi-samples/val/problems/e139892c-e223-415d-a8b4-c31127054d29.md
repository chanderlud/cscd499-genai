**Spec:**
- Write a function that checks the availability of the Windows biometric verifier (Windows Hello).
- Return a custom enum `BiometricStatus` with variants for each possible availability state.
- Map the Windows API's `UserConsentVerifierAvailability` values to your enum variants.
- Use the `Unknown` variant for any unhandled availability values from the API.
- Propagate any errors from the async operation as `windows::core::Result` errors.

**Constraints:**
- Must use `windows::Security::Credentials::UI::UserConsentVerifier::CheckAvailabilityAsync`.
- Must properly await the async operation using `.join()`.
- Must convert the result to a custom enum, not raw integers.
- Must map all known availability states from the API to specific enum variants.

**Signature:**
```rust
pub fn check_biometric_availability() -> windows::core::Result<BiometricStatus>
```

**Example:**
```rust
match check_biometric_availability()? {
    BiometricStatus::Available => println!("Biometrics ready"),
    BiometricStatus::DeviceNotPresent => println!("No biometric device"),
    BiometricStatus::NotConfigured => println!("Biometrics not set up"),
    BiometricStatus::DisabledByPolicy => println!("Biometrics disabled by policy"),
    BiometricStatus::DeviceBusy => println!("Biometric device busy"),
    BiometricStatus::Unknown => println!("Unknown status"),
}
```
