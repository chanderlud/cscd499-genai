**Spec:** Encrypt data for the current user and decrypt it back, then verify that the decrypted data matches the original. Do not use heap allocation (no `Vec`, no `Box`, etc.) in the implementation.
**Constraints:** Use `CryptProtectData` / `CryptUnprotectData` with the `CRYPTPROTECT_UI_FORBIDDEN` flag and an empty description string. Support optional "entropy" bytes (may be empty). The function must not allocate on the heap (no `Vec`, no `Box`, etc.). The Win32 functions allocate memory for the blobs, which must be freed using `LocalFree`.
**Signature:**
```rust
pub fn dpapi_roundtrip_verify(data: &[u8], entropy: &[u8]) -> Result<()>;
```
**Example:**
```rust
dpapi_roundtrip_verify(b"secret", b"pepper")?;
```
