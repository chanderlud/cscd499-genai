**Spec:** Implement two functions for converting between Rust strings and Windows wide strings (UTF-16 with null terminator) using Win32 API functions. The first function converts a valid UTF-8 string to a null-terminated UTF-16 vector. The second function converts a null-terminated UTF-16 string pointer back to a Rust string. Both functions must fail on invalid input sequences rather than performing lossy conversion.

**Constraints:**
- The first function must accept a `&str` and return a `Vec<u16>` containing the UTF-16 representation with an appended null terminator.
- The first function must use `MultiByteToWideChar` with `CP_UTF8` and `MB_ERR_INVALID_CHARS` to ensure strict UTF-8 validation.
- The second function must accept a `*const u16` pointer to a null-terminated UTF-16 string and return a `String`.
- The second function must use `WideCharToMultiByte` with `CP_UTF8` and `WC_ERR_INVALID_CHARS` to ensure strict UTF-16 validation.
- Both functions must return `windows::core::Result` and propagate Win32 errors.
- The second function must return an error if the pointer is null.

**Signature:**
```rust
fn to_wide_null(s: &str) -> windows::core::Result<Vec<u16>>;
fn from_wide_ptr(ptr: *const u16) -> windows::core::Result<String>;
```

**Example:**
```rust
let original = "Hello, Windows!";
let wide = to_wide_null(original)?;
let converted = from_wide_ptr(wide.as_ptr())?;
assert_eq!(original, converted);
```
