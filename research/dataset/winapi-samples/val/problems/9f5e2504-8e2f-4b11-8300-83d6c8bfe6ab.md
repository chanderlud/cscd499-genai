**Spec:**
- Create a topmost overlay window that can toggle between click-through (mouse pass-through) and interactive (mouse capturing) modes via a function call.
- Implement dynamic opacity control that adjusts the window's transparency level independently of its click-through state.
- The window must maintain its position and size when toggling modes and support real-time opacity adjustments without flickering.

**Constraints:**
- Use `WS_EX_LAYERED` and `SetLayeredWindowAttributes` for transparency management.
- Toggle `WS_EX_TRANSPARENT` extended style to switch between click-through and interactive modes.
- Store current mode and opacity state internally; provide getter functions for both.
- Ensure window remains topmost and does not activate when toggling modes or adjusting opacity.
- Handle `WM_NCHITTEST` appropriately for each mode (return `HTTRANSPARENT` for click-through, `HTCLIENT` for interactive).

**Signature:**
```rust
pub fn create_toggle_overlay(class_name: &str, width: i32, height: i32) -> windows::core::Result<HWND>;
pub fn set_click_through(hwnd: HWND, click_through: bool) -> windows::core::Result<()>;
pub fn set_overlay_opacity(hwnd: HWND, opacity: u8) -> windows::core::Result<()>;
pub fn is_click_through(hwnd: HWND) -> windows::core::Result<bool>;
pub fn get_overlay_opacity(hwnd: HWND) -> windows::core::Result<u8>;
```

**Example:**
```rust
let hwnd = create_toggle_overlay("ToggleOverlay", 800, 600).unwrap();
set_overlay_opacity(hwnd, 180).unwrap(); // 70% opacity
set_click_through(hwnd, false).unwrap(); // Make interactive
// Later...
set_click_through(hwnd, true).unwrap(); // Make click-through
set_overlay_opacity(hwnd, 128).unwrap(); // 50% opacity
let is_click = is_click_through(hwnd).unwrap();
let opacity = get_overlay_opacity(hwnd).unwrap();
```
