1) Peek Next Key Virtual-Key Code

**Spec:** Write a function that peeks the message queue for the next key message without removing it and returns the corresponding virtual-key code.

**Constraints:**
- Use `PeekMessageW` with `PM_NOREMOVE` to check for key messages in the range `WM_KEYFIRST` to `WM_KEYLAST`
- Extract the scancode and extended flag from the message's `lParam`
- Combine scancode and extended flag into an extended scancode value
- Use `GetKeyboardLayout` to obtain the current keyboard layout
- Use `MapVirtualKeyExW` with `MAPVK_VSC_TO_VK_EX` to map the extended scancode to a virtual-key code
- Return `None` if no key message is available, otherwise return the virtual-key code

**Signature:**
```rust
fn peek_next_key_vkey() -> Result<Option<u32>>
```

**Example:**
```rust
use windows::core::Result;

fn main() -> Result<()> {
    if let Some(vkey) = peek_next_key_vkey()? {
        println!("Next key virtual-key code: 0x{:04X}", vkey);
    } else {
        println!("No key messages in queue");
    }
    Ok(())
}
```
