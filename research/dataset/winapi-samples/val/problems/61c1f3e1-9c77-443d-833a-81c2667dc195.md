**Spec:** Create an `IShellItem` from a path and write its `SIGDN_FILESYSPATH` into a provided buffer without heap allocation.
**Constraints:** 
- Use `SHCreateItemFromParsingName` and `IShellItem::GetDisplayName`.
- Do not allocate on the heap (no `Vec`, `String`, `PathBuf`, etc.).
- The input path must be an absolute path to an existing file or directory and must be no longer than 260 characters (including the null terminator).
- The output buffer must be a mutable slice of `u16`. The function writes the null-terminated wide string into the buffer and returns the number of characters (not including the null terminator) written.
- If the output buffer is too small, the function returns an error.
**Signature:**
```rust
pub fn shellitem_filesystem_path(path: &Path, output: &mut [u16]) -> Result<usize>;
```
**Example:**
```rust
let mut buffer = [0u16; 260];
let len = shellitem_filesystem_path(Path::new(r"C:\tmp\a.txt"), &mut buffer)?;
// Now buffer[..len] contains the wide characters of the path, and buffer[len] is 0.
```
