**Spec:** Write a function that determines the location of a keyboard key from its scancode only if the current process holds a specified privilege. The function should first verify the privilege by enumerating the current process's token privileges, then (if the privilege is present) convert the scancode to a key location using the provided keyboard layout.

**Constraints:**
- Must use `OpenProcessToken`, `GetTokenInformation`, and `LookupPrivilegeNameW` to enumerate privileges
- Must use `MapVirtualKeyExW` with `MAPVK_VSC_TO_VK_EX` to convert scancode to virtual key
- Must handle dynamic buffer allocation for token information
- Must distinguish between left/right modifier keys and numpad/standard navigation keys
- Return `Err` if the required privilege is not held by the process

**Signature:**
```rust
fn get_key_location_if_privileged(
    scancode: u16,
    hkl: windows::Win32::UI::Input::KeyboardAndMouse::HKL,
    required_privilege: &str,
) -> windows::core::Result<KeyLocation>
```

**Example:**
```rust
let hkl = unsafe { windows::Win32::UI::Input::KeyboardAndMouse::GetKeyboardLayout(0) };
// Only returns location if process has SeDebugPrivilege
let location = get_key_location_if_privileged(0xE01C, hkl, "SeDebugPrivilege")?;
println!("Key location: {:?}", location);
```
