**Spec:** Watch a directory and return when you observe “a file was created” (or time out). The function must not perform any heap allocation.
**Constraints:** 
- Use `CreateFileW` on directory with `FILE_FLAG_BACKUP_SEMANTICS | FILE_FLAG_OVERLAPPED` + `ReadDirectoryChangesW`
- No heap allocation allowed (no `Vec`, `String`, `OsString`, etc.)
- Return the file name in a fixed-size buffer of 260 `u16` characters (typical MAX_PATH)
- The directory path must be provided as a null-terminated wide string slice
**Signature:**
```rust
pub fn wait_for_create(dir: &[u16], timeout_ms: u32) -> windows::core::Result<Option<([u16; 260], usize)>>;
```
**Example:**
```rust
let dir_wide = [67u16, 58, 92, 116, 109, 112, 92, 119, 97, 116, 99, 104, 0]; // "C:\tmp\watch"
let result = wait_for_create(&dir_wide, 2000)?;
```
