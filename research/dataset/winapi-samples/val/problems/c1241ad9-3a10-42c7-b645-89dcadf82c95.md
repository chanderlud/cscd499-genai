**Spec:** Write a function that returns an iterator over processor time percentages for the first CPU core using the PDH API. The iterator should yield a new value every second and automatically clean up resources when dropped.

**Constraints:**
- Use the `windows` crate for error handling and string types (PCWSTR, HRESULT, Error, Result)
- Use the PDH API via FFI (linking to `pdh` and declaring the necessary functions)
- The function must return an iterator that yields `Result<f64, windows::core::Error>`
- The iterator must collect data every second (blocking the current thread)
- The counter path must be exactly `"\\Processor(0)\\% Processor Time"`
- The iterator must properly close the PDH query when dropped
- All PDH API calls must return errors via the Result type

**Signature:**
```rust
fn processor_time_iter() -> windows::core::Result<impl Iterator<Item = windows::core::Result<f64>>>
```

**Example:**
```rust
let iter = processor_time_iter()?;
for result in iter.take(3) {
    match result {
        Ok(value) => println!("{:.2}", value),
        Err(e) => eprintln!("Error: {}", e),
    }
}
// Possible output:
// 12.34
// 15.67
// 8.91
```
