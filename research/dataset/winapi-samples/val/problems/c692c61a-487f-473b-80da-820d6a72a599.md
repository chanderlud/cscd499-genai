**Spec:** Implement a function that derives cryptographic keys from passwords using the PBKDF2 algorithm via Windows BCrypt API. The function uses HMAC with the specified hash algorithm and sets iteration count and salt as BCrypt properties. It supports configurable hash algorithms (SHA-256, SHA-384, SHA-512), iteration counts, and salt values.

**Constraints:**
- Use BCrypt's PBKDF2 implementation via `BCryptKeyDerivation` with HMAC flag
- Must handle both empty and non-empty salt values
- Must validate iteration count (minimum 1000)
- Must support variable output key lengths
- Must properly manage BCrypt algorithm and key handles using RAII patterns
- Return `windows::core::Result<Vec<u8>>` with derived key bytes

**Signature:**
```rust
fn pbkdf2_derive(
    password: &[u8],
    salt: Option<&[u8]>,
    iterations: u32,
    hash_algorithm_id: &[u8],
    key_length: usize,
) -> windows::core::Result<Vec<u8>>
```

**Example:**
```rust
// Derive a 32-byte key using PBKDF2-HMAC-SHA256
let password = b"correct horse battery staple";
let salt = b"unique_salt_value";
let derived_key = pbkdf2_derive(
    password,
    Some(salt),
    100_000,
    BCRYPT_SHA256_ALGORITHM,
    32,
)?;
assert_eq!(derived_key.len(), 32);
```
