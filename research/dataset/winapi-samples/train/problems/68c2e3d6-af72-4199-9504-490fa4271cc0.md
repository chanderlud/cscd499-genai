**Spec:**
- Import the RSA private key using the provided modulus, public exponent, prime1, and prime2 (all as byte slices in big-endian, potentially with leading zeros).
- Remove any leading zero bytes from the provided components before building the key blob.
- Convert the components to little-endian byte order for the Windows CNG key blob format.
- Compute the SHA-256 hash of the input message.
- Sign the hash using the imported RSA private key with PKCS1 padding.
- Return the signature as a byte vector.

**Constraints:**
- Use the Windows CNG API (`BCryptOpenAlgorithmProvider`, `BCryptImportKeyPair`, `BCryptCreateHash`, `BCryptHashData`, `BCryptFinishHash`, `BCryptSignHash`).
- The key must be imported as `BCRYPT_RSAPRIVATE_BLOB` using a properly formatted `BCRYPT_RSAKEY_BLOB` structure.
- The hash algorithm must be SHA-256 (`BCryptOpenAlgorithmProvider` with `BCRYPT_SHA256_ALGORITHM`).
- The signature algorithm must be PKCS1 (`BCRYPT_PAD_PKCS1`).
- Properly manage and clean up all CNG handles (algorithm providers, key, hash) using RAII patterns.
