N) Get System Metrics with DPI Fallback

**Spec:** Implement a function that retrieves system metrics (like screen dimensions) for a specified DPI value, using the DPI-aware `GetSystemMetricsForDpi` API when available on the system, and falling back to the standard `GetSystemMetrics` API otherwise.

**Constraints:**
- Must use the `windows` crate for Win32 API bindings.
- Must dynamically load `GetSystemMetricsForDpi` from `user32.dll` at runtime.
- Must cache the function pointer after first successful load using `OnceLock`.
- Must handle the case where `GetSystemMetricsForDpi` is unavailable by falling back to `GetSystemMetrics`.
- The fallback must be automatic and transparent to the caller.

**Signature:**
```rust
fn get_system_metrics_for_dpi(nindex: SYSTEM_METRICS_INDEX, dpi: u32) -> i32
```

**Example:**
```rust
let dpi = 96;
let width = get_system_metrics_for_dpi(SM_CXSCREEN, dpi);
let height = get_system_metrics_for_dpi(SM_CYSCREEN, dpi);
println!("Screen dimensions at {} DPI: {}x{}", dpi, width, height);
```
