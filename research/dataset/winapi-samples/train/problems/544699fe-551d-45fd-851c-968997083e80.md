**Spec:**
- Open the target process by its PID with `PROCESS_VM_READ` and `PROCESS_QUERY_INFORMATION` access.
- Use `CreateToolhelp32Snapshot` to get a snapshot of the process's modules.
- Enumerate the modules using `Module32FirstW` and `Module32NextW` to find the one matching the given module name (case-insensitive).
- Read the entire module's memory (from its base address to base address + size) into a buffer.
- Write the raw bytes to a file named `<module_name>.dump` in the current directory using the Win32 `WriteFile` function.
- Return `Ok(())` on success, or an appropriate Windows error on failure.

**Constraints:**
- Must handle cases where the process or module is not found.
- Must use `ReadProcessMemory` for reading and the Win32 `WriteFile` function for writing.
- The function must be safe and not crash on invalid inputs.

**Signature:**
```rust
fn dump_remote_module(pid: u32, module_name: &str) -> windows::core::Result<()>
```

**Example:**
```rust
// Dump notepad.exe's module to "notepad.exe.dump"
dump_remote_module(1234, "notepad.exe")?;
```
