**Spec:** Schedule a waitable timer with a completion routine; enter an alertable wait using `WaitForSingleObjectEx`; return whether the APC ran. The solution must not use any custom `Drop` implementations (RAII) and must close the timer handle manually in all code paths.
**Constraints:**
- Use `CreateWaitableTimerW` and `SetWaitableTimer` with a completion routine.
- Use `WaitForSingleObjectEx` for the alertable wait (do not use `SleepEx`).
- Do not use any `Drop` implementations (including guards) for resource management.
- Close the timer handle with `CloseHandle` in every exit path (success and error).
- The APC runs in the same thread, so a non-atomic `bool` is sufficient for the fired flag.
**Signature:**
```rust
pub fn apc_timer_fires(due_ms: i64, timeout_ms: u32) -> windows::core::Result<bool>;
```
**Example:**
```rust
let ok = apc_timer_fires(50, 1000)?;
```
