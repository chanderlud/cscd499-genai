**Spec:**
- Create a toast notification with a title, message, and action button
- The action button must use protocol activation with a custom URI scheme
- The toast should use the "ToastGeneric" template for Windows 10+ compatibility
- Return `Ok(())` on success or propagate Windows API errors

**Constraints:**
- Must initialize COM with `COINIT_APARTMENTTHREADED` before calling Windows APIs
- Must properly escape XML content in the toast template
- Must use the provided `app_id` parameter for notification targeting
- The URI scheme must be properly formatted as a protocol activation argument

**Signature:**
```rust
pub fn show_protocol_toast(app_id: &str, title: &str, message: &str, button_text: &str, protocol_uri: &str) -> windows::core::Result<()>
```

**Example:**
```rust
// Show a toast that opens app settings when button is clicked
show_protocol_toast(
    "MyApp.1.0.0",
    "Settings Updated",
    "Your preferences have been saved successfully",
    "Open Settings",
    "myapp://settings"
)?;
```
