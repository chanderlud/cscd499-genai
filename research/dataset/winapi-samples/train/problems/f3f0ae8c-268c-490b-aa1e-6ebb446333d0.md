**Spec:**
- Define `HeapRegion` struct with fields `base_address: usize` and `size: usize`.
- Define `HeapInfo` struct with fields `address: usize` and `regions: Vec<HeapRegion>`.
- The function must take a process handle and a process ID, and return a list of `HeapInfo`.
- Use `Heap32First`/`Heap32Next` for heap enumeration and `Heap32ListFirst`/`Heap32ListNext` for heap list traversal.
- The function must properly close all snapshot handles (using RAII or explicit close).
- Only heap lists with the `HF32_DEFAULT` flag are processed.

**Constraints:**
- Process handle must have `PROCESS_QUERY_INFORMATION` and `PROCESS_VM_READ` access rights (though the function does not use the handle, it is required by the signature).
- Must handle processes with no heaps gracefully (return empty vector).
- Must use `TH32CS_SNAPHEAPLIST` snapshot flag with the target process ID.

**Signature:**
```rust
pub fn enumerate_process_heaps(handle: windows::Win32::Foundation::HANDLE, process_id: u32) -> windows::core::Result<Vec<HeapInfo>>
```

**Example:**
```rust
let process = open_process("target.exe")?;
let heaps = enumerate_process_heaps(process.handle, process.id)?;
for heap in heaps {
    println!("Heap at 0x{:X} with {} regions", heap.address, heap.regions.len());
}
```
