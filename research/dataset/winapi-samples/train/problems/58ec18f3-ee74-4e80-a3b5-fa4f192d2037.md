**Spec:** Write a function that computes the SHA-256 hash of streaming input data using only FIPS-approved algorithms. The function must:
1. Use `BCryptGetFipsAlgorithmMode` to verify the system is in FIPS mode. Return an error if not.
2. Open the SHA-256 algorithm provider with `BCryptOpenAlgorithmProvider` using `BCRYPT_SHA256_ALGORITHM` and a custom constant `BCRYPT_FIPS_ALGORITHM_FLAG` (value 0x00000001).
3. Create a hash object with `BCryptCreateHash`.
4. Hash the provided data chunks sequentially with `BCryptHashData` as they arrive from an iterator, skipping empty chunks.
5. Retrieve the 32-byte digest with `BCryptFinishHash`.
6. Clean up the hash handle with `BCryptDestroyHash` and the algorithm handle with `BCryptCloseAlgorithmProvider`, ignoring any errors during cleanup.

**Constraints:**
- Must use `BCryptGetFipsAlgorithmMode` to check FIPS mode and return an error if not enabled.
- Must define `BCRYPT_FIPS_ALGORITHM_FLAG` as 0x00000001 since it's not exposed by the windows crate.
- Must use `BCRYPT_SHA256_ALGORITHM` with `BCRYPT_FIPS_ALGORITHM_FLAG` in `BCryptOpenAlgorithmProvider`.
- Must use `BCryptCreateHash`, `BCryptHashData`, and `BCryptFinishHash` to compute the hash.
- Must check the return status of each BCrypt function and return an error if it is not `STATUS_SUCCESS`.
- Must properly close algorithm and hash handles using `BCryptCloseAlgorithmProvider` and `BCryptDestroyHash` in all cases (success and error).
- Input data must be processed as a stream of byte slices via an iterator without collecting all chunks into a single contiguous allocation.
- The function must handle arbitrarily many chunks with total size limited only by available memory.

**Signature:**
```rust
fn compute_fips_sha256_streaming<'a>(chunks: impl IntoIterator<Item = &'a [u8]>) -> windows::core::Result<[u8; 32]>
```

**Example:**
```rust
let chunks: Vec<&[u8]> = vec![b"Hello, ", b"FIPS!"];
let hash = compute_fips_sha256_streaming(chunks)?;
println!("SHA-256: {:02x?}", hash);
```
