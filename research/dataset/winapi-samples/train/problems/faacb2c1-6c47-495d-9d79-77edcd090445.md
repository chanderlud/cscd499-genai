**Spec:** Write a function that enumerates all committed memory regions in a remote process and logs their details (base address, size, protection flags) to a file. The function should find the target process by name using toolhelp snapshots, open it with appropriate access rights, walk its virtual address space using `VirtualQueryEx`, and write a formatted report to the specified output path.

**Constraints:**
- Must use `CreateToolhelp32Snapshot` with `TH32CS_SNAPPROCESS` and `Process32FirstW`/`Process32NextW` to find the process ID by name.
- Must use `OpenProcess` with `PROCESS_QUERY_INFORMATION | PROCESS_VM_READ` access.
- Must use `VirtualQueryEx` to iterate through memory regions.
- Only include regions with `State` equal to `MEM_COMMIT`.
- Output file must contain one line per region in the format: `0x{base_address:X} - 0x{end_address:X} ({size} bytes) Protection: {protection_flags:X}`.
- Must properly close all handles (process and snapshot) and file resources.
- Return `Ok(())` on success, or an appropriate Windows error on failure.

**Signature:**
```rust
fn log_process_memory_regions(process_name: &str, output_path: &str) -> windows::core::Result<()>
```

**Example:**
```rust
// Log memory regions of notepad.exe to a file
log_process_memory_regions("notepad.exe", "memory_report.txt")?;
```
