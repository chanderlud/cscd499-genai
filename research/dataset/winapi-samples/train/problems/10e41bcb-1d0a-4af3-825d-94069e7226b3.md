**Spec:**
- Create a function `update_toast` that updates an existing toast notification
- The function must accept an application ID, a notification tag, and a new `Toast` instance
- Use the `ToastNotificationManager` to get the toast notifier for the application
- Create a new toast notification from the provided `Toast` instance
- Set the tag on the new notification to the provided tag
- Show the new notification, which will replace any existing notification with the same tag
- Return `Ok(())` on success or an appropriate Windows error

**Constraints:**
- Must use COM initialization (`COINIT_APARTMENTTHREADED`)
- Must properly handle HSTRING conversions for Windows Runtime
- Must work on Windows 10 and later
- Must preserve the original notification's tag when updating
- Should handle cases where the notification doesn't exist gracefully

**Signature:**
```rust
pub fn update_toast(app_id: &str, tag: &str, new_toast: Toast) -> windows::core::Result<()>
```

**Example:**
```rust
// Create initial toast
let toast1 = Toast::new("MyApp")
    .title("Original Notification")
    .text1("This is the original content");

// Show it with a tag
show_toast_with_tag("MyApp", "update-demo", toast1)?;

// Create updated toast
let toast2 = Toast::new("MyApp")
    .title("Updated Notification")
    .text1("This content has been updated!");

// Update the existing notification
update_toast("MyApp", "update-demo", toast2)?;
```
