**Spec:** Write a function that exports an ECC public key (ECDSA or ECDH) from a CNG key handle into the standard uncompressed point format (0x04 || X || Y). The function should query the key's algorithm name to determine the key type, query the key length, export the key blob using the appropriate blob type, validate the blob structure (magic number and key length), and return the concatenated coordinates.

**Constraints:**
- Must handle both ECDSA and ECDH public keys
- Must dynamically determine required buffer sizes
- Must validate the exported blob's magic number matches the expected public key type
- Must ensure the exported coordinates are of equal length
- Must not leak memory on error paths

**Signature:**
```rust
fn export_ecc_public_key_to_uncompressed_point(
    key_handle: BCRYPT_KEY_HANDLE,
) -> windows::core::Result<Vec<u8>>
```

**Example:**
```rust
// Assume `ecdsa_key_handle` is a valid BCRYPT_KEY_HANDLE for an ECDSA P-256 public key
let uncompressed_point = export_ecc_public_key_to_uncompressed_point(ecdsa_key_handle)?;
assert_eq!(uncompressed_point[0], 0x04); // Uncompressed point marker
assert_eq!(uncompressed_point.len(), 1 + 32 + 32); // For P-256: 1 + 32 + 32 bytes
```
