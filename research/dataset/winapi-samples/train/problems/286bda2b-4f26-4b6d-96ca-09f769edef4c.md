**Spec:**
- Implement a function that derives a cryptographic key from a password and salt using PBKDF2 with BCrypt.
- The function must support SHA-256, SHA-384, and SHA-512 hash algorithms.
- Use the BCrypt API's `BCryptDeriveKeyPBKDF2` function for the derivation.
- Return the derived key as a byte vector.
- The function validates that `key_length` fits in a `u32` and that `iterations` is non-zero, returning an error if not.

**Constraints:**
- Must use the provided `BCRYPT_ALG_HANDLE` values for the hash algorithms (SHA256, SHA384, SHA512).
- Must handle Windows API errors and return `windows::core::Result<Vec<u8>>`.
- The function should be safe to call from multiple threads.
- Do not use any external crates beyond `windows` for the Win32 API.
- The function must validate that `key_length` fits in a `u32` and that `iterations` is non-zero, returning an error if not.

**Signature:**
```rust
pub fn pbkdf2_derive(
    alg_handle: BCRYPT_ALG_HANDLE,
    password: &[u8],
    salt: &[u8],
    iterations: u32,
    key_length: usize,
) -> windows::core::Result<Vec<u8>>
```

**Example:**
```rust
let password = b"my_secure_password";
let salt = b"random_salt_value";
let iterations = 10000;
let key_length = 32; // 256 bits

let derived_key = pbkdf2_derive(
    windows::Win32::Security::Cryptography::BCRYPT_SHA256_ALG_HANDLE,
    password,
    salt,
    iterations,
    key_length,
)?;

assert_eq!(derived_key.len(), key_length);
```
