**Spec:**
- Open the target process with `PROCESS_ALL_ACCESS` rights.
- Allocate memory in the remote process with `PAGE_READWRITE` protection.
- Write the DLL path as a wide string (UTF-16) into the allocated memory.
- Obtain the address of `LoadLibraryW` from `kernel32.dll`.
- Create a remote thread in the target process with `LoadLibraryW` as its start routine and the allocated memory as its parameter.
- Wait for the remote thread to finish execution before cleaning up.
- Return `Ok(())` on success, or an appropriate Windows error on failure.

**Constraints:**
- Must use `windows::Win32::System::Threading::CreateRemoteThreadEx` for thread creation.
- Must properly close all handles (process and thread) on both success and error paths.
- The DLL path must be passed as a null-terminated wide string.
- The function must be safe to call from multiple threads concurrently.

**Signature:**
```rust
fn inject_dll(process_id: u32, dll_path: &str) -> windows::core::Result<()>
```

**Example:**
```rust
// Inject a DLL into a running Notepad instance
let pid = 1234; // Replace with actual Notepad PID
inject_dll(pid, "C:\\path\\to\\my.dll")?;
println!("DLL injected successfully");
```
