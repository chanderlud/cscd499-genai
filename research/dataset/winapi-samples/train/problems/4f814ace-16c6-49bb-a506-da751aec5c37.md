**Spec:** Write a function that derives a cryptographic key from input key material using the HMAC-based key derivation function (HKDF) with BCrypt. The function should support SHA-256, SHA-384, and SHA-512 hash algorithms.

**Constraints:**
- Use BCrypt's HMAC functionality via `BCryptCreateHash` with the appropriate HMAC algorithm handles
- Implement the HKDF extract-then-expand paradigm as defined in RFC 5869
- The function must accept a hash algorithm identifier (SHA-256, SHA-384, or SHA-512)
- Handle variable-length output key material (OKM) up to 255 times the hash length
- Return the derived key as a byte vector
- Use RAII guards to ensure proper cleanup of BCrypt handles

**Signature:**
```rust
fn hkdf_derive(
    algorithm: PCWSTR,
    ikm: &[u8],
    salt: Option<&[u8]>,
    info: Option<&[u8]>,
    okm_length: usize,
) -> windows::core::Result<Vec<u8>>
```

**Example:**
```rust
let ikm = b"input_key_material";
let salt = b"unique_salt";
let info = b"application_context";
let derived_key = hkdf_derive(
    BCRYPT_SHA256_ALGORITHM,
    ikm,
    Some(salt),
    Some(info),
    32, // 256-bit key
)?;
assert_eq!(derived_key.len(), 32);
```
