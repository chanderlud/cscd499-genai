**Spec:**
- Write a function that checks if Windows is running in FIPS mode and verifies that a specified cryptographic algorithm is FIPS-approved.
- The function should return a tuple of two booleans: (fips_mode_enabled, algorithm_is_fips_approved).
- Use the Windows CNG API to query FIPS mode status and algorithm properties.

**Constraints:**
- Must use `BCryptGetFipsAlgorithmMode` to check system FIPS status.
- Must use `BCryptOpenAlgorithmProvider` with `BCRYPT_FIPS_ALGORITHM_FLAG` to test algorithm FIPS compliance.
- Must properly close algorithm handles using `BCryptCloseAlgorithmProvider`.
- The algorithm identifier should be passed as a string parameter (e.g., "SHA256", "AES").
- Since `BCRYPT_FIPS_ALGORITHM_FLAG` is not defined in the `windows` crate, it must be defined locally as a constant with value `0x00000002`.
- The function must return an error if `BCryptGetFipsAlgorithmMode` fails or if `BCryptOpenAlgorithmProvider` fails with an error other than `ERROR_INVALID_PARAMETER` (0x80070057). In the case of `ERROR_INVALID_PARAMETER`, the algorithm is considered not FIPS-approved.

**Signature:**
```rust
fn check_fips_algorithm_support(algorithm_id: &str) -> windows::core::Result<(bool, bool)>
```

**Example:**
```rust
let (fips_enabled, sha256_fips) = check_fips_algorithm_support("SHA256")?;
println!("FIPS mode: {}, SHA256 FIPS-approved: {}", fips_enabled, sha256_fips);
```
