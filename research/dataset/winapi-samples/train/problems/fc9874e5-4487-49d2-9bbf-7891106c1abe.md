TITLE: FIPS Algorithm Check

**Spec:** Write a function that checks whether a specific cryptographic algorithm is FIPS-approved on the current Windows system. The function should:
1. Verify if Windows is running in FIPS mode using `BCryptGetFipsAlgorithmMode`
2. If FIPS mode is enabled, open the specified algorithm provider using `BCryptOpenAlgorithmProvider`. For hash algorithms (SHA256, SHA384, SHA512) use the `BCRYPT_ALG_HANDLE_HMAC_FLAG` flag; for other algorithms (AES, RSA, ECDSA) use default flags.
3. Check the algorithm's FIPS compliance by querying the "FIPSAlgorithm" property using `BCryptGetProperty`
4. Return `true` only if both conditions are met (system in FIPS mode AND the algorithm's FIPS property is non-zero). Otherwise, return `false`.

**Constraints:**
- Must use Windows CNG API (`BCryptGetFipsAlgorithmMode`, `BCryptOpenAlgorithmProvider`, `BCryptGetProperty`, `BCryptCloseAlgorithmProvider`)
- Handle algorithm names: "SHA256", "SHA384", "SHA512", "AES", "RSA", "ECDSA"
- For hash algorithms (SHA256, SHA384, SHA512) use the `BCRYPT_ALG_HANDLE_HMAC_FLAG` when opening the algorithm provider
- Return `false` if system is not in FIPS mode, regardless of algorithm properties
- Properly close algorithm handles using `BCryptCloseAlgorithmProvider`

**Signature:**
```rust
fn is_algorithm_fips_approved(algorithm_name: &str) -> windows::core::Result<bool>
```

**Example:**
```rust
// Check if SHA256 is FIPS-approved
let sha256_approved = is_algorithm_fips_approved("SHA256")?;
println!("SHA256 FIPS approved: {}", sha256_approved);

// Check if AES is FIPS-approved
let aes_approved = is_algorithm_fips_approved("AES")?;
println!("AES FIPS approved: {}", aes_approved);
```
