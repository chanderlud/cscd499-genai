**Spec:** Write a function that reads a source file in fixed-size chunks, reverses the order of those chunks, and writes them to a destination file. The function should handle files larger than available memory by processing one chunk at a time.

**Constraints:**
- Use `CreateFileW` with `OPEN_EXISTING` for reading and `CREATE_ALWAYS` for writing.
- Process the file in chunks of exactly 4096 bytes, except possibly the last chunk.
- The reversal must be at the chunk level, not byte level (i.e., the first chunk read becomes the last chunk written).
- Use `SetFilePointerEx` to navigate the source file for reading chunks in reverse order.
- Ensure proper error handling and resource cleanup via `CloseHandle`.

**Signature:**
```rust
pub fn reverse_file_chunks(src_path: &str, dst_path: &str) -> windows::core::Result<()>
```

**Example:**
```rust
// Assuming "input.txt" contains "ABCDEFGHIJ" (10 bytes)
reverse_file_chunks("input.txt", "reversed.txt")?;
// With 4-byte chunks: ["ABCD", "EFGH", "IJ"]
// "reversed.txt" will contain "EFGHIJABCD"
```
