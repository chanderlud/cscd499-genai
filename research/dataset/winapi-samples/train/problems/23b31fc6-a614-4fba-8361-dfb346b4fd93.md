**Spec:**
- Create a topmost, layered, click-through overlay window with `WS_EX_LAYERED | WS_EX_TOPMOST | WS_EX_TRANSPARENT`
- Implement a timer that periodically (every 500ms) finds the window at the overlay's center point
- Sample a 10x10 pixel region from the target window's client area at the overlay's center
- Calculate the average RGB color of the sampled region
- Update the overlay's color-key transparency using `SetLayeredWindowAttributes` with `LWA_COLORKEY` to match the calculated color
- Use a fallback color of black (0x00000000) when sampling fails or target window is invalid

**Constraints:**
- Must handle windows with different background colors and themes
- Should gracefully handle cases where the underlying window has no solid background (use fallback color black)
- Must not cause performance issues with frequent sampling (500ms interval, 10x10 pixel sample)
- Should work with both classic and modern Windows applications
- Must properly release all device contexts and handles after sampling
- Overlay must remain click-through and not interfere with underlying window interaction

**Signature:**
```rust
pub fn create_adaptive_overlay(width: i32, height: i32) -> windows::core::Result<windows::Win32::Foundation::HWND>
```

**Example:**
```rust
let hwnd = create_adaptive_overlay(200, 200)?;
// Overlay will now automatically adapt its color-key to match
// the background of whatever window is beneath it
std::thread::sleep(std::time::Duration::from_secs(10));
// After 10 seconds, the overlay will have sampled and adapted
// to underlying windows multiple times
```
