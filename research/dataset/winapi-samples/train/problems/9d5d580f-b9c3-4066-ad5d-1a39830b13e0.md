**Spec:** Watch a directory for file system changes using `ReadDirectoryChangesW`. The function must use a fixed-size buffer (provided by the caller) and must not reallocate or resize it. If a notification exceeds the buffer capacity, the function must return an error and stop watching. Each raw `FILE_NOTIFY_INFORMATION` record buffer is sent through the returned receiver.

**Constraints:**
- Buffer must be a fixed-size array (no heap allocation for the notification buffer)
- No buffer resizing or reallocation allowed
- If `ERROR_NOTIFY_ENUM_DIR` occurs due to buffer overflow, return error immediately
- Must use overlapped I/O with a dedicated thread
- Path must exist and be a directory

**Signature:**
```rust
pub fn watch_dir_fixed_buffer<const N: usize>(
    path: &Path,
    recursive: bool,
    buffer: &mut [u8; N],
) -> windows::core::Result<std::sync::mpsc::Receiver<Vec<u8>>>;
```

**Example:**
```rust
let mut buffer = [0u8; 4096];
let rx = watch_dir_fixed_buffer(Path::new(r"C:\tmp"), true, &mut buffer)?;
```
