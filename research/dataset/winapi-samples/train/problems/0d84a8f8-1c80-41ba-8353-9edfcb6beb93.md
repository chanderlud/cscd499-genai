**Spec:**
- Create a function that computes and verifies the hash of a file's contents using the Windows BCrypt API
- The function should read a file in chunks, update the hash incrementally, then compare against an expected hash value
- Return `Ok(true)` if hashes match, `Ok(false)` if they don't match, or propagate any I/O or cryptographic errors
- The algorithm provider must be opened with the `BCRYPT_HASH_REUSABLE_FLAG` for efficiency

**Constraints:**
- Must use BCrypt's hash functions (`BCryptOpenAlgorithmProvider`, `BCryptCreateHash`, `BCryptHashData`, `BCryptFinishHash`, `BCryptDestroyHash`, `BCryptCloseAlgorithmProvider`)
- Must handle files larger than available memory by processing in 4KB chunks
- Must properly clean up cryptographic handles and file handles using RAII patterns
- Must not load entire file into memory at once
- Must use Win32 file APIs (`CreateFileW`, `ReadFile`) with sequential scan optimization

**Signature:**
```rust
fn verify_file_hash(
    file_path: &std::path::Path,
    expected_hash: &[u8],
    algorithm: &Algorithm
) -> windows::core::Result<bool>
```

**Example:**
```rust
let file_path = std::path::Path::new("document.txt");
let expected_hash = hex::decode("5e884898da28047151d0e56f8dc6292773603d0d6aabbdd62a11ef721d1542d8")?;
let algorithm = Algorithm::new("SHA256")?;

match verify_file_hash(file_path, &expected_hash, &algorithm) {
    Ok(true) => println!("File integrity verified"),
    Ok(false) => println!("File has been modified"),
    Err(e) => println!("Verification failed: {}", e),
}
```
