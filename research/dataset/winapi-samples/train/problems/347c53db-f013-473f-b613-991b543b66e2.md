**Spec:** Implement two functions that use the Windows BCrypt API to perform authenticated encryption and decryption with AES in Galois/Counter Mode (GCM). The encryption function must return both ciphertext and authentication tag. The decryption function must verify the tag and return plaintext only if authentication succeeds.

**Constraints:**
- Use BCrypt's `BCryptEncrypt` and `BCryptDecrypt` functions
- Support 128-bit, 192-bit, and 256-bit AES keys
- Nonce (IV) must be 12 bytes (96 bits) as recommended for GCM
- Tag length must be 16 bytes (128 bits)
- Input plaintext/ciphertext length must not exceed 2^32 - 2 bytes
- Return `windows::core::Error` on invalid arguments (wrong nonce length, key length, or data length) or authentication failure during decryption

**Signature:**
```rust
fn aes_gcm_encrypt(
    key: &[u8],
    nonce: &[u8],
    plaintext: &[u8],
    aad: Option<&[u8]>,
) -> windows::core::Result<(Vec<u8>, [u8; 16])>

fn aes_gcm_decrypt(
    key: &[u8],
    nonce: &[u8],
    ciphertext: &[u8],
    tag: &[u8; 16],
    aad: Option<&[u8]>,
) -> windows::core::Result<Vec<u8>>
```

**Example:**
```rust
let key = [0x42; 32]; // 256-bit key
let nonce = [0x01; 12];
let plaintext = b"secret message";

let (ciphertext, tag) = aes_gcm_encrypt(&key, &nonce, plaintext, None).unwrap();
let decrypted = aes_gcm_decrypt(&key, &nonce, &ciphertext, &tag, None).unwrap();
assert_eq!(decrypted, plaintext);
```
