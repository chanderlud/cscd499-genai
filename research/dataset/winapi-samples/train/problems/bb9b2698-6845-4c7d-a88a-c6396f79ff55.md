**Spec:**
- Implement a function that imports a Diffie-Hellman private key into CNG using the BCrypt API.
- The function receives the DH parameters: prime modulus `p`, generator `g`, public key `y`, and private key `x` as byte slices.
- Construct the appropriate `BCRYPT_DH_KEY_BLOB` structure and call `BCryptImportKeyPair` with `BCRYPT_DH_PRIVATE_BLOB`.
- Return an owned key handle on success.

**Constraints:**
- All input byte slices (`p`, `g`, `y`, `x`) must have the same length (the key size in bytes).
- The function must return an error if the input slices have different lengths.
- The function must correctly set the `BCRYPT_DH_PRIVATE_MAGIC` value in the blob header.
- The blob must be constructed in the exact layout required by the Windows API: header followed by `p`, `g`, `y`, and `x` concatenated in that order.
- Use `windows::core::Owned` to manage the returned key handle.

**Signature:**
```rust
fn import_dh_private_key(
    alg_handle: BCRYPT_ALG_HANDLE,
    p: &[u8],
    g: &[u8],
    y: &[u8],
    x: &[u8],
) -> windows::core::Result<windows::core::Owned<BCRYPT_KEY_HANDLE>>
```
