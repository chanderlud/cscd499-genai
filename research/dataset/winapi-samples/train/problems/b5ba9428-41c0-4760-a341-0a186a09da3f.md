**Spec:**  
Write a function `get_registry_string_noalloc` that opens a specified Windows registry key and reads a string value (REG_SZ) from it, returning the value as a Rust `String`. The function must not perform any heap allocation except for the final `String` returned.

**Constraints:**  
- Use the `windows` crate for Win32 API bindings.
- The function must open the key with `KEY_READ` access.
- It must handle the two-step registry query pattern: first call to get required buffer size, second call to read data.
- The registry key must be properly closed even if errors occur during value reading.
- Return `Err` if the value type is not `REG_SZ`, if any Win32 API call fails, or if the registry path, value name, or data exceed fixed stack buffer limits.
- All intermediate buffers (for wide strings and registry data) must be stack-allocated with fixed maximum sizes: 260 characters for the path and value name (including null terminator), and 2048 characters (4096 bytes) for the registry data.
- No heap allocation (e.g., `Vec`, `String` except for the return value) is allowed during the operation.

**Signature:**  
```rust
fn get_registry_string_noalloc(hive: HKEY, path: &str, value_name: &str) -> windows::core::Result<String>
```

**Example:**  
```rust
let product_name = get_registry_string_noalloc(
    HKEY_LOCAL_MACHINE,
    "SOFTWARE\\Microsoft\\Windows NT\\CurrentVersion",
    "ProductName"
)?;
println!("Windows Product Name: {}", product_name);
```
