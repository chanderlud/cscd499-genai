**Spec:** Perform the same SSPI NTLM handshake and EncryptMessage/DecryptMessage roundtrip as the original challenge, but without any heap allocation for intermediate buffers. All handshake tokens, encrypted data, and security trailers must use stack-allocated fixed-size arrays sized to 4096 bytes.

**Constraints:** 
- No heap allocation (`Vec`, `Box`, etc.) for intermediate buffers during handshake or encryption/decryption
- Use fixed-size arrays of 4096 bytes for all intermediate buffers
- Must validate that input plaintext fits within the fixed buffer sizes (considering security trailer overhead)
- Must use `windows::core::Result<T>` for all fallible operations
- Must handle the complete NTLM handshake in memory without sockets
- Must query `SECPKG_ATTR_SIZES` for security trailer size after context establishment
- Must use `QuerySecurityPackageInfoW` to get maximum token size (though not used for buffer sizing)

**Signature:**
```rust
pub fn sspi_ntlm_seal_roundtrip_stack(plaintext: &[u8]) -> Result<Vec<u8>>;
```
