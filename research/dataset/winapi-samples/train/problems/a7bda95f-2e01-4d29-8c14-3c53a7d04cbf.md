**Spec:** Create a Win32 window containing a ListBox that displays the primary monitor's device name, dimensions, and default DPI (hardcoded as 96) as its first item, followed by nine additional numbered items. The ListBox must be subclassed to handle keyboard navigation and selection: Escape exits the application, Return selects the current item (printing its text to standard output), and Up/Down arrows navigate items.

**Constraints:**
- Use `MonitorFromPoint` with origin (0,0) and `MONITOR_DEFAULTTOPRIMARY` to get the primary monitor handle
- Use `GetMonitorInfoW` to retrieve monitor information into a `MONITORINFOEXW` structure
- Format the primary monitor information as a string (e.g., "Primary: \\.\DISPLAY1, 1920x1080, DPI:96") and add it as the first ListBox item
- Populate the ListBox with nine additional items ("Item 1" through "Item 9")
- Implement window subclassing via `SetWindowLongPtrW`/`GetWindowLongPtrW` with `GWL_WNDPROC`
- Store the original window procedure using `GWLP_USERDATA`
- Handle `WM_KEYDOWN` and `WM_KEYUP` messages in the subclass procedure
- Call the original window procedure for default handling of Up/Down navigation

**Signature:**
```rust
fn main() -> windows::core::Result<()>;
extern "system" fn window_proc(hwnd: HWND, msg: u32, wparam: WPARAM, lparam: LPARAM) -> LRESULT;
extern "system" fn list_box_proc(hwnd: HWND, msg: u32, wparam: WPARAM, lparam: LPARAM) -> LRESULT;
```

**Example:**
```rust
// The ListBox should contain:
//   "Primary: \\.\DISPLAY1, 1920x1080, DPI:96"

//   "Item 2"
//   ...
//   "Item 9"
// Pressing Escape closes the window.
// Pressing Return prints the selected item's text to standard output (e.g., "Selected: Item 1").
// Up/Down arrows navigate the list.
```
