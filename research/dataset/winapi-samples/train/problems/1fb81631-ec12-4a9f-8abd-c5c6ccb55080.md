**Spec:**
- Implement a function that encrypts plaintext using AES-GCM with a provided key and nonce.
- The function must use BCrypt to perform the encryption and return the ciphertext and authentication tag.
- The algorithm provider for AES must be opened and then set to GCM mode by setting the chaining mode property.
- The function must accept plaintext and additional authenticated data (AAD) as iterators over bytes, collecting them internally.

**Constraints:**
- Use `windows::core::Result<T>` for error handling.
- The key must be 256 bits (32 bytes) and the nonce must be 96 bits (12 bytes) as required by AES-GCM.
- The function should handle the algorithm provider initialization internally, using thread-local storage for efficiency.
- The plaintext and AAD parameters must be generic over iterators that yield `u8`.

**Signature:**
```rust
pub fn encrypt_aes_gcm_iter<I, J>(
    key: &[u8; 32],
    nonce: &[u8; 12],
    plaintext: I,
    aad: J,
) -> windows::core::Result<(Vec<u8>, [u8; 16])>
where
    I: IntoIterator<Item = u8>,
    J: IntoIterator<Item = u8>,
```

**Example:**
```rust
let key = [0u8; 32];
let nonce = [0u8; 12];
let plaintext = b"secret message".iter().copied();
let aad = b"additional data".iter().copied();
let (ciphertext, tag) = encrypt_aes_gcm_iter(&key, &nonce, plaintext, aad).unwrap();
```
