**Spec:** Open a file and return its final DOS-style path. The function must not allocate any heap memory for intermediate buffers (the wide string for the input path and the buffer for the final path). Use stack-allocated arrays of fixed maximum size (1024 characters). Return an error if the input path or final path exceeds this limit.
**Constraints:** Use `CreateFileW` with `FILE_FLAG_BACKUP_SEMANTICS` and `FILE_ATTRIBUTE_NORMAL`, and share modes `FILE_SHARE_READ | FILE_SHARE_WRITE | FILE_SHARE_DELETE`. Use `GetFinalPathNameByHandleW` with `FILE_NAME_NORMALIZED` and `VOLUME_NAME_DOS`. No heap allocation for intermediate buffers (use fixed-size stack arrays of 1024 characters). Return an error if the input path or final path is too long for the fixed buffer.
**Signature:**
```rust
pub fn final_path(path: &Path) -> windows::core::Result<PathBuf>;
```
**Example:**
```rust
let p = final_path(Path::new(r"C:\tmp\a.txt"))?;
```
