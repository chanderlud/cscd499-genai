**Spec:**
- Create a thread that opens a file and then closes the handle, demonstrating safe handle management within the thread.
- Convert a Win32 error code to an HRESULT and return it as a `windows::core::Result`.

**Constraints:**
- Use `CreateFileW` to open a file and `CloseHandle` to close it.
- The file handle must be created and closed entirely within the thread to ensure thread safety.
- Use `CreateThread` to create a thread and pass the file path as a `String` by converting it to a raw pointer and then reconstructing it in the thread procedure.
- The thread procedure must be an `unsafe extern "system"` function.

**Signature:**
```rust
/// Convert an OsStr to a wide null-terminated string.
fn wide_null(s: &OsStr) -> Vec<u16>;

/// Open a file handle in a thread-safe manner.
fn open_file_in_thread(path: &OsStr) -> windows::core::Result<windows::Win32::Foundation::HANDLE>;

/// Thread worker function that opens a file and then closes the handle.
fn file_reader_thread(path: String) -> windows::core::Result<()>;

/// Simulate a Win32 error and convert it to HRESULT.
fn check_access_denied() -> windows::core::Result<()>;

/// Main function that creates a thread and waits for it.
fn main() -> windows::core::Result<()>;

/// Thread procedure that reconstructs the String and calls file_reader_thread.
unsafe extern "system" fn thread_proc(param: *mut std::ffi::c_void) -> u32;
```

**Example:**
```rust
// Run the main function which creates a thread to open a file and then close the handle.
// It also demonstrates error conversion.
if let Err(e) = main() {
    eprintln!("Error: {}", e);
}
```
