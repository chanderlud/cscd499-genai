**Spec:**
- Generate an RSA key pair using BCrypt with configurable key size (must be 1024, 2048, or 4096 bits)
- Hash a message using SHA-256 via BCrypt
- Sign the hash with the generated private key using RSA PKCS#1 v1.5 padding
- Verify the signature using the corresponding public key
- Return whether verification succeeds

**Constraints:**
- Use BCrypt for all cryptographic operations (key generation, hashing, signing, verification)
- Properly manage BCrypt handles with RAII patterns (similar to source snippet)
- Handle errors via `windows::core::Result<T>`
- Support key sizes of exactly 1024, 2048, and 4096 bits (return error for other sizes)

**Signature:**

    message: &[u8],
    key_size: u32,
