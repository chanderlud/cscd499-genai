1) Check for More Character Messages in the Queue

**Spec:** Write a function that checks whether the next pending message in the message queue for a specified window is a character message (WM_CHAR or WM_SYSCHAR), without removing the message from the queue.

**Constraints:**
- Use the `PeekMessageW` Win32 API with `PM_NOREMOVE` to inspect the queue.
- Only consider messages in the range `WM_KEYFIRST` to `WM_KEYLAST`.
- Return `true` if the next message is `WM_CHAR` or `WM_SYSCHAR`, otherwise `false`.
- Handle the case where no message is available by returning `false`.

**Signature:**
```rust
fn has_more_char_messages(hwnd: HWND) -> Result<bool>
```

**Example:**
```rust
let hwnd = HWND::default(); // In practice, a valid window handle
let more_coming = has_more_char_messages(hwnd)?;
println!("More character messages coming: {}", more_coming);
```
