1) Map HMONITOR to Monitor Friendly Name

**Spec:** Implement two functions: one that generates a lookup table mapping GDI device names to monitor friendly names for all active displays, and another that uses that table to retrieve the friendly name for a given `HMONITOR` handle.

**Constraints:**
- Use Win32 display configuration APIs (`GetDisplayConfigBufferSizes`, `QueryDisplayConfig`, `DisplayConfigGetDeviceInfo`) to build the lookup table.
- Use `GetMonitorInfoW` to obtain the GDI device name for the provided `HMONITOR`.
- The lookup table must be built for active paths only and with virtual mode awareness.
- The lookup function should return `None` if the monitor information cannot be retrieved or the GDI device name is not found in the table.

**Signature:**
```rust
pub fn generate_gdi_to_monitor_name_lookup() -> Result<HashMap<String, String>>;
pub fn monitor_name_from_hmonitor(lookup: &mut HashMap<String, String>, handle: isize) -> Option<String>;
```

**Example:**
```rust
let mut lookup = generate_gdi_to_monitor_name_lookup()?;
// Obtain an HMONITOR from MonitorFromWindow or similar
let hmonitor: isize = /* ... */;
if let Some(name) = monitor_name_from_hmonitor(&mut lookup, hmonitor) {
    println!("Monitor name: {}", name);
}
```
