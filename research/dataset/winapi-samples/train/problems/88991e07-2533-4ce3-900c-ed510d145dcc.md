**Spec:**
- Implement a function that derives a cryptographic key from a password and salt using PBKDF2-HMAC-SHA256 via the Windows BCrypt API
- The function should accept a password, salt, iteration count, and desired output key length
- Return the derived key as a vector of bytes

**Constraints:**
- Must use BCrypt's PBKDF2 implementation (BCryptDeriveKeyPBKDF2)
- Must handle all BCrypt resource cleanup properly (close algorithm provider handle)
- Must validate input parameters: non-empty password, non-zero iterations, non-zero key_length
- Must return E_INVALIDARG (0x80070057) for invalid input parameters
- Must return appropriate errors for BCrypt API failures

**Signature:**
```rust
fn derive_key_pbkdf2(
    password: &[u8],
    salt: &[u8],
    iterations: u32,
    key_length: usize,
) -> windows::core::Result<Vec<u8>>
```

**Example:**
```rust
let password = b"my_secure_password";
let salt = b"random_salt_value";
let iterations = 100_000;
let key_length = 32; // 256-bit key

let derived_key = derive_key_pbkdf2(password, salt, iterations, key_length)?;
assert_eq!(derived_key.len(), key_length);
```
