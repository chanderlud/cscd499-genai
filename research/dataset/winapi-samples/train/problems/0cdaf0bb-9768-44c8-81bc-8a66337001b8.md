**Spec:**
- Write a function that sets the home directory and profile path for an existing local user account.
- Use `NetUserSetInfo` with `USER_INFO_1` structure to update the user's home directory (`usri1_home_dir`) and script path (`usri1_script_path`).
- The function retrieves the current user info first, modifies only the specified fields (if an `Option` is `Some`), then applies the changes.

**Constraints:**
- Must handle the case where the user does not exist (return appropriate error).
- Must preserve all other existing user attributes (password, flags, etc.).
- Use `windows::core::Result<()>` for error handling.
- Server name parameter is optional (use `None` for local machine).
- Passing `None` for `home_dir` or `profile_path` leaves that field unchanged.
- To clear the home directory or profile path, pass an empty string (`Some("")`).

**Signature:**
```rust
fn set_user_paths(
    server: Option<&str>,
    username: &str,
    home_dir: Option<&str>,
    profile_path: Option<&str>,
) -> windows::core::Result<()>
```

**Example:**
```rust
// Set both home directory and profile path for user "john"
set_user_paths(None, "john", Some("C:\\Users\\john"), Some("C:\\Profiles\\john"))?;

// Leave home directory unchanged, but set profile path
set_user_paths(None, "john", None, Some("C:\\Profiles\\john"))?;

// Clear home directory by setting it to an empty string
set_user_paths(None, "john", Some(""), Some("C:\\Profiles\\john"))?;
```
