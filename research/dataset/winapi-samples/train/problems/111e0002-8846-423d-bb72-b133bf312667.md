**Spec:**
- Accept a process handle, base address, region size, and new protection constant.
- Use `VirtualProtectEx` to change the protection of the memory region.
- Return the previous protection flags on success.

**Constraints:**
- Must handle errors and return a `windows::core::Result`.
- The address and size must be valid for the target process.
- The new protection must be a valid `PAGE_PROTECTION_FLAGS` constant.

**Signature:**
```rust
pub fn change_memory_protection(
    handle: windows::Win32::Foundation::HANDLE,
    address: u32,
    size: usize,
    new_protection: windows::Win32::System::Memory::PAGE_PROTECTION_FLAGS,
) -> windows::core::Result<windows::Win32::System::Memory::PAGE_PROTECTION_FLAGS>
```

**Example:**
```rust
// Assume we have a valid process handle from somewhere (e.g., from `OpenProcess`).
let handle: windows::Win32::Foundation::HANDLE = ...; // obtained elsewhere
let old_prot = change_memory_protection(
    handle,
    0x12345678,
    4096,
    windows::Win32::System::Memory::PAGE_READWRITE,
)?;
println!("Old protection: {:?}", old_prot);
```
