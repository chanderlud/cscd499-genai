**Spec:**
- Write a function that checks biometric verifier availability and, if available, requests verification with a custom message (without a window handle).
- Return the verification result enum on success, or an error if the verifier is unavailable.

**Constraints:**
- Use `UserConsentVerifier::CheckAvailabilityAsync` and `UserConsentVerifier::RequestVerificationAsync` (non-window version).
- Wait for async operations using `.join()`.
- Return `windows::core::Result<UserConsentVerificationResult>`.

**Signature:**
```rust
pub fn request_biometric_verification(message: &str) -> windows::core::Result<UserConsentVerificationResult>
```

**Example:**
```rust
let result = request_biometric_verification("Authenticate to continue");
match result {
    Ok(verification) => println!("Verification: {:?}", verification),
    Err(e) => println!("Verifier unavailable: {}", e),
}
```
