**Spec:** Implement HKDF key derivation using BCrypt primitives. The function should take an input key material (IKM), optional salt, optional info string, and desired output length, then derive a cryptographic key using the HMAC-based Extract-and-Expand paradigm.

**Constraints:**
- Use BCrypt HMAC functions for both extract and expand phases
- Support SHA-256, SHA-384, and SHA-512 as underlying hash algorithms
- Handle optional salt (use zero-filled hash-length buffer if not provided)
- Handle optional info (use empty buffer if not provided)
- Return derived key material of exact requested length
- Must properly handle HMAC key size requirements (keys longer than block size get hashed)

**Signature:**
```rust
fn hkdf_derive(
    alg_handle: BCRYPT_ALG_HANDLE,
    ikm: &[u8],
    salt: Option<&[u8]>,
    info: Option<&[u8]>,
    output_len: usize,
) -> windows::core::Result<Vec<u8>>
```

**Example:**
```rust
let ikm = b"input key material";
let salt = b"optional salt";
let info = b"context info";
let derived = hkdf_derive(BCRYPT_SHA256_ALG_HANDLE, ikm, Some(salt), Some(info), 32)?;
assert_eq!(derived.len(), 32);
```
