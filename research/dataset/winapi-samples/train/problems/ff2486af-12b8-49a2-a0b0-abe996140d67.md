**Spec:**
- Find the process ID by name by enumerating processes and comparing the image path using `K32EnumProcesses` and `K32GetModuleFileNameExW`.
- Open the target process with `PROCESS_QUERY_INFORMATION` and `PROCESS_VM_READ` access rights.
- Dynamically load `NtQueryInformationProcess` from ntdll.dll to retrieve the `PROCESS_BASIC_INFORMATION` structure containing the PEB address.
- Read a partial remote PEB structure to obtain the `ProcessParameters` pointer.
- Read a partial `RTL_USER_PROCESS_PARAMETERS` structure to get the `ImagePathName` (a `UNICODE_STRING`).
- Read the actual image path string from the remote process memory using the `Buffer` and `Length` fields.
- Return the image path as a `String`.

**Constraints:**
- Must use `ReadProcessMemory` for all remote memory reads.
- Must properly close the process handle after reading (using RAII/scopeguard).
- Must handle errors at each step (process not found, memory read failures, etc.) and return a `windows::core::Result`.
- The function should work for any accessible process on the system.

**Signature:**
```rust
pub fn get_remote_process_image_path(process_name: &str) -> windows::core::Result<String>
```

**Example:**
```rust
let image_path = get_remote_process_image_path("notepad.exe")?;
println!("Notepad image path: {}", image_path);
// Output might be: "C:\Windows\System32\notepad.exe"
```
