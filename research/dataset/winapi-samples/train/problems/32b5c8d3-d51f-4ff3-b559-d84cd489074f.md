- Implement `encrypt` method that takes plaintext and associated data, returns ciphertext and authentication tag
- Implement `decrypt` method that takes ciphertext, associated data, and tag, returns plaintext or error if authentication fails
- Use AES-256-GCM with 12-byte nonces and 16-byte authentication tags

**Constraints:**
- Use BCrypt API for all cryptographic operations
- Properly manage BCrypt handles with RAII patterns
- Handle all error cases and return appropriate `windows::core::Result` errors
- Ensure nonce is generated internally for encryption (callers don't provide it)
