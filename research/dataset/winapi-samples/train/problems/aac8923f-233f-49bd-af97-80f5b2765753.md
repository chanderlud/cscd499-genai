**Spec:**
- Implement HKDF key derivation (RFC 5869) using BCrypt's HMAC operations
- Provide both HKDF-Extract and HKDF-Expand functions
- Support SHA-256 as the underlying hash algorithm
- Handle salt and info parameters correctly per RFC 5869 specification

**Constraints:**
- Use BCrypt API for all cryptographic operations (BCryptOpenAlgorithmProvider, BCryptCreateHash, BCryptHashData, BCryptFinishHash, BCryptGetProperty)
- Must properly manage BCrypt algorithm and hash handles with RAII patterns
- Output length for HKDF-Expand must not exceed 255×HashLen (RFC 5869 limit)
- Return clear errors for invalid parameters or BCrypt failures

**Signature:**
```rust
struct HkdfSha256 {
    // Internal fields for BCrypt algorithm handle
}

impl HkdfSha256 {
    fn new() -> windows::core::Result<Self>;
    
    fn extract(&self, salt: &[u8], ikm: &[u8]) -> windows::core::Result<Vec<u8>>;
    
    fn expand(&self, prk: &[u8], info: &[u8], length: usize) -> windows::core::Result<Vec<u8>>;
}
```

**Example:**
```rust
let hkdf = HkdfSha256::new()?;
let salt = b"optional salt";
let ikm = b"input key material";

// Extract phase
let prk = hkdf.extract(salt, ikm)?;

// Expand phase
let info = b"application context";
let derived_key = hkdf.expand(&prk, info, 32)?;

println!("Derived {} bytes", derived_key.len());
```
