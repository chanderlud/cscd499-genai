**Spec:**
- Implement `wrap_key` that encrypts a 128/192/256-bit AES key using a wrapping AES key of the same size
- Use BCrypt's `BCryptEncrypt` with the `BCRYPT_BLOCK_PADDING` flag and handle the 8-byte IV automatically
- Return the wrapped key (which will be 8 bytes longer than the input key)

**Constraints:**
- The wrapping key must be provided as raw bytes
- The key to wrap must be exactly 128, 192, or 256 bits (16, 24, or 32 bytes)
- Both keys must be the same length
- Use `BCRYPT_AES_ALGORITHM` with `BCRYPT_CHAIN_MODE_ECB` (AES Key Wrap uses ECB internally)
- Handle the default IV (0xA6A6A6A6A6A6A6A6) automatically

**Signature:**
```rust
fn wrap_key(wrapping_key: &[u8], key_to_wrap: &[u8]) -> windows::core::Result<Vec<u8>>
```

**Example:**
```rust
let wrapping_key = [0x00, 0x01, 0x02, 0x03, 0x04, 0x05, 0x06, 0x07,
                    0x08, 0x09, 0x0A, 0x0B, 0x0C, 0x0D, 0x0E, 0x0F];
let key_to_wrap = [0x00, 0x11, 0x22, 0x33, 0x44, 0x55, 0x66, 0x77,
                   0x88, 0x99, 0xAA, 0xBB, 0xCC, 0xDD, 0xEE, 0xFF];
let wrapped = wrap_key(&wrapping_key, &key_to_wrap).unwrap();
assert_eq!(wrapped.len(), key_to_wrap.len() + 8);
```
