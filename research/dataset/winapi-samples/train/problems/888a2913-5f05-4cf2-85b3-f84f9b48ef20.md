**Spec:** Write a function that merges two files by alternating fixed-size chunks from each into a new output file. Read a chunk from the first file, then a chunk from the second file, and repeat until both files are exhausted. If one file ends before the other, append the remaining chunks from the longer file. Return an error if any file operation fails or if chunk_size is zero.

**Constraints:**
- Use Windows API (`CreateFileW`, `ReadFile`, `WriteFile`, `CloseHandle`) for all file operations.
- Chunk size must be a positive (non-zero) `u32`.
- Open source files with `GENERIC_READ`, `FILE_SHARE_READ`, and `OPEN_EXISTING`.
- Create the output file with `GENERIC_WRITE`, `FILE_SHARE_NONE`, `CREATE_ALWAYS`, and `FILE_ATTRIBUTE_NORMAL`.
- Return an error if any file operation fails.

**Signature:**
```rust
pub fn merge_files_alternating(
    path1: &str,
    path2: &str,
    output_path: &str,
    chunk_size: u32,
) -> windows::core::Result<()>
```

**Example:**
```rust
// Assuming "a.txt" contains "ABCDEFGH" and "b.txt" contains "1234"
merge_files_alternating("a.txt", "b.txt", "merged.txt", 2)?;
// "merged.txt" now contains "AB12CD34EF"
```
