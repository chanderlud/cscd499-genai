- **Spec:** Normalize a `windows::core::Result<()>` so that a provided set of Win32 error codes are treated as success. Only Win32 errors (identified by facility code 7) are considered for normalization; other errors are preserved unchanged. The set is given as a fixed-size array of `u32` error codes.
- **Constraints:** 
  - Must not allocate heap memory (no `Vec`, `String`, etc.).
  - Must use only the provided array (no other collections).
  - Must compare error codes as integers, not strings.
  - Preserve all other failures unchanged.
- **Signature:**
  ```rust
  pub fn ok_if_error_in_list<const N: usize>(result: windows::core::Result<()>, error_codes: &[u32; N]) -> windows::core::Result<()>;
  ```
- **Example:**
  ```rust
  use windows::core::{Error, HRESULT};
  use windows::Win32::Foundation::{ERROR_ALREADY_EXISTS, ERROR_FILE_EXISTS};

  let codes = [ERROR_ALREADY_EXISTS.0, ERROR_FILE_EXISTS.0];
  let r = Err(Error::from_hresult(HRESULT::from_win32(ERROR_ALREADY_EXISTS.0)));
  assert!(ok_if_error_in_list(r, &codes).is_ok());
  ```
