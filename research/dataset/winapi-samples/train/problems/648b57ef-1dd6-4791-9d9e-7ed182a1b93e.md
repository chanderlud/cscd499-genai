1) Implement LPARAM coordinate packing and unpacking functions

**Spec:** Implement three functions to pack two 16-bit signed integers into an LPARAM value and extract them back, replicating the behavior of the Win32 `GET_X_LPARAM`, `GET_Y_LPARAM`, and `MAKELPARAM` macros.

**Constraints:**
- Use the `windows` crate and the `LPARAM` type from `windows::Win32::Foundation`
- Functions must correctly handle negative 16-bit integers using two's complement representation
- Functions must be marked `#[inline]` and use `#[allow(non_snake_case)]` to match Win32 macro naming conventions
- The packing function must place the first parameter in the low 16 bits and the second in the high 16 bits

**Signature:**
```rust
#[allow(non_snake_case)]
#[inline]
pub fn GET_X_LPARAM(lparam: LPARAM) -> i16;

#[allow(non_snake_case)]
#[inline]
pub fn GET_Y_LPARAM(lparam: LPARAM) -> i16;

#[allow(non_snake_case)]
#[inline]
pub fn MAKELPARAM(x: i16, y: i16) -> LPARAM;
```

**Example:**
```rust
use windows::Win32::Foundation::LPARAM;

let x: i16 = 100;
let y: i16 = -200;
let lparam = MAKELPARAM(x, y);
assert_eq!(x, GET_X_LPARAM(lparam));
assert_eq!(y, GET_Y_LPARAM(lparam));
```
