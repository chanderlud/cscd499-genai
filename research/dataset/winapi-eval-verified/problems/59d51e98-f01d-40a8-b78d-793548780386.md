15. SRWLock + ConditionVariable bounded queue stress test

* **Spec:** Implement a bounded multi-producer/multi-consumer queue using only SRW locks + condition variables; run a stress scenario and return all consumed items. The function should produce `n_items` items (the integers 0..n_items) and then have `producers` producer threads and `consumers` consumer threads. The queue has a fixed `capacity`. The function returns a vector of the consumed items (in the order they were consumed, but the order is not specified and may be non-deterministic). If any of the parameters are invalid (capacity == 0, producers == 0, or consumers == 0), the function returns an error.
* **Constraints:** `InitializeSRWLock`, `AcquireSRWLockExclusive/Shared`, `InitializeConditionVariable`, `SleepConditionVariableSRW`, `WakeConditionVariable/WakeAllConditionVariable`. No Rust `Mutex/Condvar`.
* **Signature:**

```text
pub fn bounded_queue_stress(n_items: u32, capacity: u32, producers: u32, consumers: u32) -> Result<Vec<u32>>;
```

* **Example:**

```text
let got = bounded_queue_stress(10_000, 64, 4, 4)?;
```
