## 6) Anonymous Pipe: Producer/Consumer Checksum

**Spec:**  
Create an anonymous pipe with `CreatePipe`. Spawn one writer thread that owns the pipe's write handle.  
The writer thread must write the full contents of `data` into the pipe using `WriteFile`, then close the write handle with `CloseHandle`.

In the calling thread, read from the pipe using `ReadFile` until EOF is reached. For this problem, EOF means `ReadFile` succeeds and reports `0` bytes read. Compute and return a checksum equal to the wrapping sum of all bytes read, i.e. the sum modulo `2^32`.

Return any OS I/O failure as `std::io::Error`.

**Important:**  
After spawning the writer thread, the caller must not retain an open write handle while reading. Otherwise EOF will not be observed.

**Constraints:**

- Must use `CreatePipe`, `ReadFile`, `WriteFile`, and `CloseHandle`
- Must not use Rust `std::process::Command` pipes
- Must not use `std::fs::File` wrappers for the pipe handles
- The writer must run on a separate thread
- The reader must loop until `ReadFile` reports EOF via `0` bytes read
- The checksum must be computed from the bytes actually read, using wrapping addition
- The implementation must write the entire input, even if multiple `WriteFile` calls are needed
- Any failure in the writer thread must be surfaced as `std::io::Result`
