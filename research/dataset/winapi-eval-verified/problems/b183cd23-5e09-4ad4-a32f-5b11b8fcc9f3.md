33) Create named shared memory (CreateFileMappingW with name)

* **Spec:** Create a named mapping for IPC shared memory.
* **Constraints:** Security/ACL matters; name collisions; size fixed unless you coordinate.
* **Signature:**

```rust
pub fn create_named_mapping(name: &str, size: usize) -> windows::core::Result<OwnedHandle>;
```

* **Example:**

```rust
let m = create_named_mapping("Local\\myshare", 4096)?;
```

**Notes:**
- The `name` must be a valid Windows object name. It is recommended to use the `Local\` prefix for session-local objects.
- The function converts the name to a wide string and null-terminates it. Names containing embedded NUL characters are invalid and will result in an error.
- The `size` must be greater than zero. The actual mapping size is rounded up to the system page size.
- The function returns an `OwnedHandle` that owns the file mapping object. When the handle is dropped, the mapping is closed.
- If a mapping with the same name already exists, the behavior is implementation-defined: it may return a handle to the existing mapping or an error. The tests cover both possibilities.
