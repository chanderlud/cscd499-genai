24. Canonical absolute path (Win32)

* **Spec:** Convert an input Windows path, which may be relative, into an absolute normalized path. Relative paths must be resolved according to Win32 rules. The function should normalize separators and collapse `.` and `..` where appropriate, but it must not resolve symlinks or junctions and must not require the path to exist.
* **Constraints:** Use `GetFullPathNameW` directly. Do **not** use `std::fs::canonicalize`. Do not implement custom path semantics. Preserve Win32 behavior for drive-relative, root-relative, UNC, device namespace, and extended-length paths.
* **Signature:**

```rust
pub fn full_path(path: &Path) -> Result<PathBuf>;
```

* **Example:**

```rust
let p = full_path(Path::new(r"..\data\file.txt"))?;
```
