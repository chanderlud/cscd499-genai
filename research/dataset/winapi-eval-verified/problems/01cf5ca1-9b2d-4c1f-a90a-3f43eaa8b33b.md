3. Named file mapping: write then read via a second mapping

* **Spec:** Create a named shared memory region, write bytes at an offset, open the same mapping again, and read back exactly `data.len()` bytes from the same offset. The function is responsible for creating the mapping first, so a fresh name should succeed.
* **Constraints:** Use `CreateFileMappingW` + `OpenFileMappingW` + `MapViewOfFile` + `UnmapViewOfFile`. Return an error if `size == 0` or if the write range is out of bounds, meaning `offset + data.len() > size`.
* **Signature:**

```rust
pub fn named_mapping_write_read(name: &str, size: usize, offset: usize, data: &[u8]) -> Result<Vec<u8>>;
```

* **Example:**

```rust
let out = named_mapping_write_read("Local\\my_map", 4096, 10, b"hello")?;
```
