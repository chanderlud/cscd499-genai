2. Named Pipe: Multi-Instance Fanout (N clients)

* **Spec:** Create a named pipe server with `instances = client_count`. Spawn `client_count` clients; each client sends its index `i` as `u32` LE. The server replies to each client with `i * 3 + 1` as `u32` LE. Return the collected replies in client index order.
* **Constraints:**

    * Use the `nMaxInstances` parameter in `CreateNamedPipeW` to set the number of instances to `client_count`.
    * Clients must connect concurrently (threads are fine).
