1. Named Pipe: Length-Prefixed Framing (byte mode)

* **Spec:** Implement a function that creates a named pipe server and a client, sends the given frames from the client to the server, and returns the lengths of the frames as reported by the server. The server reads exactly `N` frames (where `N` is the length of the `frames` slice) and replies with the list of received frame lengths (as a `u32` list encoded the same way: count + lengths). The communication uses length-prefixed framing in byte mode. The `pipe_name` parameter is the name part of the pipe (without the `\\.\pipe\` prefix).
* **Constraints:**

    * Use Win32 named pipes in **byte mode** (`PIPE_TYPE_BYTE`).
    * Must correctly handle partial reads/writes (loop until complete).
    * No external framing libraries.
    * The function must create the server and client internally (e.g., using threads) and clean up the pipe resources before returning.
