14. LockFileEx exclusive lock with contention check

* **Spec:** Acquire an exclusive lock on a byte range; on a second handle, attempt the same lock and return whether it correctly fails/blocks (your choice, but define behavior).
* **Constraints:** Use `LockFileEx` / `UnlockFileEx` and proper `OVERLAPPED` range targeting.
* **Signature:**

```rust
pub fn exclusive_lock_contended(path: &Path, offset: u64, len: u64) -> Result<bool>;
```

* **Example:**

```rust
let contended = exclusive_lock_contended(Path::new(r"C:\tmp\a.bin"), 0, 4096)?;
```
