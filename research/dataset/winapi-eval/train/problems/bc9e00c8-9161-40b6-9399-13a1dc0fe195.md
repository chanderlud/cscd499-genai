- **Spec:**
  Implement a function named `run_stage_red_light_rehearsal` for a fictional film studio safety drill.

  The function must:
    - create an unnamed **manual-reset** Windows event that starts in the **nonsignaled** state
    - perform a zero-time probe wait before signaling and record whether that probe timed out
    - spawn `worker_count` threads, where each thread waits on the same event and increments a shared counter exactly once after being released
    - wait until every worker has reported that it is ready to wait
    - signal the event exactly once
    - join every worker thread
    - perform a second zero-time probe wait after all workers have finished
    - return a report describing:
        - whether the initial probe timed out
        - how many workers were awakened
        - whether the post-signal probe observed the event as still signaled

- **Constraints:**
    - Use only the `windows` crate version `0.62.2`
    - Use the Win32 APIs `CreateEventW`, `SetEvent`, `WaitForSingleObject`, and `CloseHandle`
    - Do not use sleeps or busy-wait loops for coordination
    - The event handle must be closed exactly once, even if an error occurs
    - The wake-up primitive must be the Windows event object, not a Rust condvar or mutex-based substitute
    - The function must be deterministic and testable for both `worker_count == 0` and `worker_count > 0`
    - Assume the crate enables these features:
        - `Win32_Foundation`
        - `Win32_Security`
        - `Win32_System_Threading`

- **Signature:**
  ```rust
  #[derive(Debug, Clone, Copy, PartialEq, Eq)]
  pub struct AlarmReport {
      pub initial_probe_timed_out: bool,
      pub awakened_workers: usize,
      pub post_signal_probe_passed: bool,
  }

  pub fn run_stage_red_light_rehearsal(
      worker_count: usize,
  ) -> windows::core::Result<AlarmReport>;
  ```
- **Example:**
  ```rust
let report = run_stage_red_light_rehearsal(3).unwrap();

assert!(report.initial_probe_timed_out);
assert_eq!(report.awakened_workers, 3);
assert!(report.post_signal_probe_passed);
  ```
