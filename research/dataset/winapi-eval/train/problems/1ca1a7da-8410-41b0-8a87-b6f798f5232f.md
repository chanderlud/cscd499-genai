## 8) Dynamic Loading: Call Function via GetProcAddress

* **Spec:** Load `kernel32.dll`, resolve `GetCurrentProcessId` with `GetProcAddress`, call it through the function pointer, and return the PID.
* **Constraints:**

  * Use `LoadLibraryW`, `GetProcAddress`, `FreeLibrary`.
  * Must call through the obtained pointer (no direct `GetCurrentProcessId()` call).
  * Validate the pointer is non-null.
  * Treat the resolved symbol as a function pointer with signature `unsafe extern "system" fn() -> u32`.
  * Return an error if `LoadLibraryW` or `GetProcAddress` fails.

* **Signature:**

  ```rust
  pub fn dynamic_get_current_process_id() -> std::io::Result<u32> {
      todo!()
  }
  ```
* **Example:**

  ```rust
  let pid = dynamic_get_current_process_id()?;
  assert!(pid != 0);
  ```
