4. SSPI “NTLM” handshake in memory + EncryptMessage/DecryptMessage

* **Spec:** Build client+server SSPI contexts by exchanging tokens in memory (no sockets). Then seal (encrypt) a message with `EncryptMessage` and recover it with `DecryptMessage`, returning the recovered message.
* **Constraints:** `AcquireCredentialsHandleW("NTLM")`, `InitializeSecurityContextW`, `AcceptSecurityContext`, `QueryContextAttributes(SECPKG_ATTR_SIZES)`, `EncryptMessage`, `DecryptMessage`.
* **Signature:**

```rust
pub fn sspi_ntlm_seal_roundtrip(plaintext: &[u8]) -> Result<Vec<u8>>;
```

* **Example:**

```rust
let out = sspi_ntlm_seal_roundtrip(b"secrets, but corporate")?;
```
