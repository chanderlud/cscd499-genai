6. Shared Memory: Producer/Consumer with Events (SPSC)

* **Spec:** Create a shared memory region with a fixed-size byte buffer and two `u32` fields: `write_len` and `read_len`. Producer writes exactly one message (byte slice) into the buffer, sets `write_len`, signals `data_ready` event. Consumer waits, copies message out, sets `read_len = write_len`, signals `ack_ready`. Return the consumer’s received bytes.
* **Constraints:**

    * Use `CreateEventW` (auto-reset or manual-reset, but be consistent and correct).
    * Use `WaitForSingleObject` with a finite timeout.
    * Buffer size fixed (e.g., 4096). If input too large, return an error.
