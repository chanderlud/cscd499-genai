### Problem

* **Spec:** Implement a function that creates exactly one directory at the given path by calling the Win32 `CreateDirectoryW` API with `NULL` security attributes. It should return `Ok(())` on success and propagate the Windows error on failure. `CreateDirectoryW` only creates the final path component, so missing parent directories should still cause an error. ([Microsoft GitHub][1])
* **Constraints:** Use only the `windows` crate version `0.62.2`. Convert the Rust `&str` path into a null-terminated UTF-16 buffer before the API call. Call only `CreateDirectoryW` from Win32 for the required OS interaction. You may assume the input path has no interior `\0` characters. ([Microsoft GitHub][1])
* **Signature:**

```rust
pub fn create_directory(path: &str) -> windows::core::Result<()>;
```

* **Example:**

```rust
fn main() -> windows::core::Result<()> {
    create_directory(r"C:\Temp\api_problem_demo")?;
    Ok(())
}
```
