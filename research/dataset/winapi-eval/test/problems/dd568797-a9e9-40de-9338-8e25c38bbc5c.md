- **Spec:**
  - Define a public struct `EventStateReport` with these fields:
    - `initial_sample_signaled: bool`
    - `after_set_sample_signaled: bool`
    - `after_reset_sample_signaled: bool`
  - Implement `probe_manual_reset_event(initial_state: bool) -> windows::core::Result<EventStateReport>`.
  - The function must create an **unnamed manual-reset event** with the requested initial state.
  - It must sample the event state three times by calling `WaitForSingleObject` with a timeout of `0`:
    1. immediately after creation
    2. after calling `SetEvent`
    3. after calling `ResetEvent`
  - Interpret `WAIT_OBJECT_0` as `true` and `WAIT_TIMEOUT` as `false`.
  - If `WaitForSingleObject` returns any other value, return a Windows error.
  - The event handle must always be closed before the function returns, including on error paths.

- **Constraints:**
  - Use only the `windows` crate version `0.62.2`.
  - Use Win32 APIs directly: `CreateEventW`, `SetEvent`, `ResetEvent`, and `WaitForSingleObject`.
  - Do not use the file system.
  - Do not use environment variables.
  - Do not panic for ordinary error handling.
  - Keep the behavior deterministic and suitable for unit tests.

- **Signature:**
  ```rust
  pub fn probe_manual_reset_event(
      initial_state: bool,
  ) -> windows::core::Result<EventStateReport>;
````

* **Example:**

  ```rust
  let report = probe_manual_reset_event(false)?;
  assert_eq!(report.initial_sample_signaled, false);
  assert_eq!(report.after_set_sample_signaled, true);
  assert_eq!(report.after_reset_sample_signaled, false);
  # Ok::<(), windows::core::Error>(())
  ```
