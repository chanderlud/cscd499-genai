- **Spec:**
  - Implement `write_readonly_temp_file` that creates or overwrites a file named `<stem>.txt` inside the process temporary directory using Win32 APIs.
  - Obtain the temporary directory with `GetTempPathW`.
  - Open the target file with `CreateFileW` using `CREATE_ALWAYS`.
  - Write all bytes from `contents` with `WriteFile`, flush them with `FlushFileBuffers`, then mark the file as read-only with `SetFileAttributesW`.
  - Return the final absolute path and the total number of bytes written.

- **Constraints:**
  - Use only the `windows` crate v0.62.2.
  - Do not use `std::fs::write`, `std::fs::File`, `OpenOptions`, or other high-level file creation/writing helpers in the main implementation.
  - Convert Rust paths/strings to UTF-16 before passing them to Win32 APIs.
  - The function must succeed for empty `contents`.
  - Assume `stem` is a simple file stem with no path separators and no interior NUL characters.

- **Signature:**
  ```rust
  pub struct TempFileWriteResult {
      pub path: std::path::PathBuf,
      pub bytes_written: u32,
  }

  pub fn write_readonly_temp_file(
      stem: &str,
      contents: &[u8],
  ) -> windows::core::Result<TempFileWriteResult>
  ```

- **Example:**
  ```rust
  let result = write_readonly_temp_file("api_demo_note", b"hello")?;
  assert!(result.path.exists());
  assert_eq!(result.bytes_written, 5);
  ```
