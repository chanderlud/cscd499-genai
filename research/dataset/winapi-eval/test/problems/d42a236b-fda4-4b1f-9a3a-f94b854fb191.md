- **Spec:**
  - Implement `run_manual_reset_event_script`, which creates an unnamed **manual-reset** event object.
  - The function receives an initial signaled state and a slice of `EventAction` values.
  - For each action:
    - `Set`: call `SetEvent`, then record whether the event is currently signaled.
    - `Reset`: call `ResetEvent`, then record whether the event is currently signaled.
    - `Check`: do not modify the event, just record whether it is currently signaled.
  - Use `WaitForSingleObject(handle, 0)` as the non-blocking way to observe the current signal state.
  - Return a `Vec<bool>` containing one observation per action, in order.
  - The event handle must always be closed before the function returns.

- **Constraints:**
  - Use only the `windows` crate v0.62.2.
  - Use these Win32 APIs directly: `CreateEventW`, `SetEvent`, `ResetEvent`, `WaitForSingleObject`, and `CloseHandle`.
  - The event must be unnamed and manual-reset.
  - Do not use the file system.
  - The function must be deterministic and unit testable.

- **Signature:**
  - ```rust
    #[derive(Debug, Clone, Copy, PartialEq, Eq)]
    pub enum EventAction {
        Set,
        Reset,
        Check,
    }

    pub fn run_manual_reset_event_script(
        initial_state: bool,
        actions: &[EventAction],
    ) -> windows::core::Result<Vec<bool>>;
    ```

- **Example:**
  - ```rust
    let states = run_manual_reset_event_script(
        false,
        &[
            EventAction::Check,
            EventAction::Set,
            EventAction::Check,
            EventAction::Reset,
        ],
    )
    .unwrap();

    assert_eq!(states, vec![false, true, true, false]);
    ```
