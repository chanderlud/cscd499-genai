- **Spec:** Implement a function `roundtrip_named_manual_reset_event` that creates or opens a named manual-reset event in the local namespace, sets it to the signaled state, checks that a zero-timeout wait reports it as signaled, resets it to the nonsignaled state, checks that a zero-timeout wait now times out, closes the handle, and returns a struct describing what happened.

- **Constraints:**
  - Use only the `windows` crate v0.62.2.
  - Use Win32 APIs directly, not `std::sync`.
  - Convert the Rust event name to a null-terminated UTF-16 string for the Win32 call.
  - Reject event names that contain an interior `\0`.
  - The function must call a few Win32 APIs with straightforward parameters:
    - `CreateEventW`
    - `GetLastError`
    - `SetEvent`
    - `ResetEvent`
    - `WaitForSingleObject`
    - `CloseHandle`
  - The function must not leak the event handle.
  - The return value must include:
    - whether the named event already existed
    - whether the wait after `SetEvent` succeeded immediately
    - whether the wait after `ResetEvent` timed out

- **Signature:**
  ```rust
  pub fn roundtrip_named_manual_reset_event(
      name: &str,
  ) -> windows::core::Result<EventRoundtrip>
````

* **Example:**

  ```rust
  let snapshot = roundtrip_named_manual_reset_event(r"Local\demo.chatgpt.event")?;
  assert!(!snapshot.already_existed);
  assert!(snapshot.signaled_immediately);
  assert!(snapshot.reset_to_nonsignaled);
  # Ok::<(), windows::core::Error>(())
  ```
