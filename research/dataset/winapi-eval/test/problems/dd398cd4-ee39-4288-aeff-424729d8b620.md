- **Spec:**
    - Implement `label_current_thread(tag)` for Windows.
    - The function must fetch the current thread ID, build the description string in the exact format `tid-<thread_id>-<tag>`, set that description on the current thread with Win32, then read the description back from Windows and return the round-tripped string.

- **Constraints:**
    - Use the Win32 APIs `GetCurrentThread`, `GetCurrentThreadId`, `SetThreadDescription`, `GetThreadDescription`, and `LocalFree`.
    - Return the exact string retrieved from Windows, not just the string you constructed.
    - The implementation must be deterministic and unit-testable without sleeping, timing checks, or user interaction.
    - Assume `tag` does not contain `'\0'`.
    - Only use the `windows` crate v0.62.2.

- **Signature:**
  ```rust
  pub fn label_current_thread(tag: &str) -> windows::core::Result<String>
  ```
- **Example:**
  ```rust
use windows::core::Result;

fn demo() -> Result<()> {
    let name = label_current_thread("worker")?;
    assert!(name.ends_with("-worker"));
    Ok(())
}
  ```
