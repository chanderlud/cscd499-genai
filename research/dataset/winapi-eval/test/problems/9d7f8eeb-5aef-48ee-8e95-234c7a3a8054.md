- **Spec:**
    - Implement a function for a retro spaceship airlock console that stores its state in a legacy `.ini` file.
    - The function must use the section named by `airlock`.
    - It must read the current `cycles` counter from the file, defaulting to `0` if the key does not exist.
    - It must increment `cycles` by exactly one.
    - It must write `badge` into the key `last_badge`.
    - It must write the updated cycle count into the key `cycles` as decimal text.
    - It must flush the profile cache for that file, then read back both values through the Win32 profile APIs and return them.

- **Constraints:**
    - Use only the `windows` crate v0.62.2 for Win32 calls.
    - Use `WritePrivateProfileStringW`, `GetPrivateProfileStringW`, and `GetPrivateProfileIntW`.
    - Do not parse or mutate the `.ini` file with `std::fs` inside the target function.
    - Create the file, section, and keys on demand.
    - Return a Win32-style error if a write fails.
    - Assume the parent directory for `ini_path` already exists.
    - Assume the counter will not overflow `u32`.

- **Signature:**
  ```rust
  pub fn stamp_airlock_log(
      ini_path: &std::path::Path,
      airlock: &str,
      badge: &str,
  ) -> windows::core::Result<AirlockLog>;
  ```
- **Example:**
    ```rust
use std::path::Path;

let snapshot = stamp_airlock_log(Path::new("airlocks.ini"), "Dock-3", "badge-042")?;
assert_eq!(snapshot.cycles, 1);
assert_eq!(snapshot.last_badge, "badge-042");
# Ok::<(), windows::core::Error>(())
```
