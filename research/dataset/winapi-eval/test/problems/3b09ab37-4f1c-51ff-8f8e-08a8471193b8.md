## 07) Extract an underlying `WIN32_ERROR` when possible

* **Spec:** If the error’s HRESULT is a `FACILITY_WIN32` value, return the underlying `WIN32_ERROR`. Otherwise return `None`.
* **Constraints:** Do not hardcode individual error values. Inspect the HRESULT layout.
* **Signature:**

```rust
pub fn try_as_win32(err: &windows::core::Error) -> Option<windows::Win32::Foundation::WIN32_ERROR>;
```

* **Example:**

```rust
use windows::core::{Error, HRESULT};
use windows::Win32::Foundation::ERROR_ACCESS_DENIED;

let err = Error::from_hresult(HRESULT::from_win32(ERROR_ACCESS_DENIED.0));
assert_eq!(try_as_win32(&err), Some(ERROR_ACCESS_DENIED));
```
