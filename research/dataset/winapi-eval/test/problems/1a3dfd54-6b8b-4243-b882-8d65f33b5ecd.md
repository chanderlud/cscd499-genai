- **Spec:**
  - Implement a function `signal_event` that takes a Windows event handle and sets that event to the signaled state by calling the Windows API `SetEvent`.
  - Return `Ok(())` if the API call succeeds, or the Windows error if it fails.

- **Constraints:**
  - Use only the `windows` crate version `0.62.2`.
  - The function must call exactly one Windows API method: `SetEvent`.
  - Do not create, destroy, or wait on the event inside the function.
  - The function must be deterministic and unit testable.
  - Assume the caller provides a valid event handle in normal usage.

- **Signature:**
  ```rust
  pub fn signal_event(event: windows::Win32::Foundation::HANDLE) -> windows::core::Result<()>
````

* **Example:**

  ```rust
  use windows::Win32::System::Threading::CreateEventW;
  use windows::Win32::Foundation::HANDLE;

  let event: HANDLE = unsafe { CreateEventW(None, true, false, None)? };
  signal_event(event)?;
  ```
