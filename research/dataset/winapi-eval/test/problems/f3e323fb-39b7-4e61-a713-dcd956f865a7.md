* **Spec:**

  * Implement a function `delete_file` that deletes the file at the given path by calling the Win32 `DeleteFileW` API. ([Microsoft Learn][2])

* **Constraints:**

  * Use only the `windows` crate v0.62.2. The Win32 file-system bindings live under `windows::Win32::Storage::FileSystem`. ([Docs.rs][3])
  * Call exactly one Win32 API function: `DeleteFileW`. Its Rust binding takes a `PCWSTR` path and returns `Result<()>`. ([Microsoft GitHub][4])
  * Convert the Rust path into a null-terminated UTF-16 buffer before making the API call.
  * Do not use `std::fs::remove_file`.
  * Propagate the Windows error if the file does not exist or cannot be deleted. Microsoft documents that `DeleteFileW` fails for missing files and other deletion errors. ([Microsoft Learn][2])

* **Signature:**

  ```rust
  pub fn delete_file(path: &std::path::Path) -> windows::core::Result<()>;
  ```

* **Example:**

  ```rust
  use std::path::Path;

  fn demo() -> windows::core::Result<()> {
      delete_file(Path::new(r"C:\Temp\example.txt"))?;
      Ok(())
  }
  ```
