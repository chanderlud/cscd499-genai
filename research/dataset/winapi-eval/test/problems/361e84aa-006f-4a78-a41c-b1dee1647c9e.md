- **Spec:** Implement a function `copy_file_winapi` that copies a file from `src` to `dst` by calling the Windows API function `CopyFileW`. The function must return `Ok(())` when the copy succeeds, and an `std::io::Error` when the API call fails.

- **Constraints:**
  - Use only the `windows` crate version `0.62.2`.
  - Call exactly one Windows API function for the copy operation: `CopyFileW`.
  - Accept any path type that can be referenced as `std::path::Path`.
  - Convert Rust paths to null-terminated UTF-16 strings before passing them to the API.
  - Respect the `fail_if_exists` flag:
    - `true` means the function must fail if `dst` already exists.
    - `false` means the destination may be overwritten.
  - Do not use `std::fs::copy` for the actual copy.

- **Signature:**
  ```rust
  pub fn copy_file_winapi(
      src: impl AsRef<std::path::Path>,
      dst: impl AsRef<std::path::Path>,
      fail_if_exists: bool,
  ) -> std::io::Result<()>;
````

* **Example:**

  ```rust
  use std::fs;
  use std::path::PathBuf;

  let mut src = std::env::temp_dir();
  src.push("copy_file_winapi_src.txt");

  let mut dst = std::env::temp_dir();
  dst.push("copy_file_winapi_dst.txt");

  fs::write(&src, b"hello").unwrap();
  copy_file_winapi(&src, &dst, true).unwrap();

  assert_eq!(fs::read(&dst).unwrap(), b"hello");
  ```
