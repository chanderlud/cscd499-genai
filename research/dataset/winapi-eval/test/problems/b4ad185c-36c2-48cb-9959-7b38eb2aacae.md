- **Spec:**
  - Implement a function `current_process_id()` that returns the current process ID as a `u32`.
  - The function must call the Windows API function `GetCurrentProcessId`.
  - The returned value must be the process ID of the running process.

- **Constraints:**
  - Use only the `windows` crate version `0.62.2`.
  - Use the API from `windows::Win32::System::Threading`.
  - Do not spawn subprocesses or read environment variables to obtain the PID.
  - Keep the public function safe to call.
  - The implementation should involve exactly one Windows API call.
  - Assume the crate feature `Win32_System_Threading` is enabled.

- **Signature:**
  ```rust
  pub fn current_process_id() -> u32
````

* **Example:**

  ```rust
  fn main() {
      let pid = current_process_id();
      println!("current pid = {}", pid);
  }
  ```
