- **Spec:**
    - Implement a function that publishes a small "beacon" into a named, pagefile-backed shared memory region using the Windows API.
    - The memory region must be named `Local\\Beacon::<name>`.
    - The function must write a fixed binary layout:
        - bytes `0..4`: ASCII magic `BCN1`
        - bytes `4..8`: little-endian `u32` payload length
        - bytes `8..12`: little-endian `u32` checksum, defined as the wrapping sum of all payload bytes
        - bytes `12..(12 + payload.len())`: UTF-8 payload bytes
    - If the mapping already exists, the function must overwrite the existing contents with the new payload and zero-fill the remaining bytes in the mapping.
    - Return a struct describing the mapping name, payload length, checksum, and whether the mapping was newly created. The struct must contain the handle to the mapping.
- **Constraints:**
    - Use only the `windows` crate `v0.62.2`.
    - Use Windows file-mapping APIs backed by the system paging file, not regular files.
    - The function must reject payloads larger than `4084` bytes.
    - The mapping size must be exactly `4096` bytes.
    - The implementation must be deterministic and safe for unit tests to verify by reopening the named mapping and reading its contents.
    - The function must return `windows::core::Result<_>`.
