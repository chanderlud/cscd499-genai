1. Named Pipe: Length-Prefixed Framing (byte mode)

* **Spec:** Implement a server that reads exactly `N` frames from a named pipe. Each frame is `u32` little-endian length followed by that many bytes. Server replies with the list of received frame lengths (as `u32` list encoded the same way: count + lengths). Client sends frames and returns the decoded lengths from the server.
* **Constraints:**

    * Use Win32 named pipes in **byte mode** (`PIPE_TYPE_BYTE`).
    * Must correctly handle partial reads/writes (loop until complete).
    * No external framing libraries.
